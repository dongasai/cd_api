## 二、计费单位换算说明

### 2.1 四种计费单位对比

```
┌─────────────────────────────────────────────────────────────────────┐
│                   Coding Plan 计费单位换算                           │
└─────────────────────────────────────────────────────────────────────┘

1. API请求次数 (阿里云百炼、火山方舟、无问芯穹)
   ├─── 1次用户提问 = 后台触发 5-30次模型调用
   ├─── 1次调用 = 1次API请求
   └──   例: 1200次/5小时 ≈ 40-240次用户提问

2. Prompt次数 (MiniMax、智谱GLM、摩尔线程)
   ├─── 1次Prompt = 1次用户提问
   ├─── 智谱GLM: 每次Prompt预计调用模型 15-20 次
   ├──   智谱GLM: 本质Token计费, 官网显示为估算值
   ├──   MiniMax: 1次Prompt ≈ 1200-1600次API请求
   └──   摩尔线程: 每次Prompt预计调用模型 15-20 次

3. Token计量 (Kimi)
   ├─── 按输入输出Token计费
   ├─── 仅统计未命中缓存的Token
   ├──   缓存命中率直接影响实际额度
   └──   官网展示的"Prompt次数"为估算值，实际按Token消耗

4. 模型消耗倍数 (参考):
   ├─── 基础模型 (GPT-3.5等)    │ 1x
   ├──   标准模型 (GPT-4o等)     │ 2x
   ├──   高级模型 (Claude 3.5)   │ 2-3x
   └──   推理模型 (o1系列)       │ 5-10x
```

### 4.2 选择核心原则

```
1. 不盲目追求"低价": 
   低价套餐往往存在额度陷阱，优先看"实际可用额度"而非表面价格

2. 匹配自身使用场景: 
   轻度用户别买重度套餐，重度用户别贪低价入门档

3. 优先薅首月优惠: 
   新用户可先体验首月低价套餐，测试后再决定长期订阅

4. 避开隐藏条款: 
   充值前查看是否支持退订、是否禁止企业使用、额度是否结转
```
