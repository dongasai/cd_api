# 模型别名功能设计文档

## 功能概述

模型别名功能允许为 AI 模型配置多个命名别名，主要用于**渠道路由扩展**场景。别名是纯粹的字符串数组，不需要在模型列表中存在对应记录。当请求某个模型时，系统自动使用该模型的所有别名名称一起搜索渠道，最大化渠道匹配能力。

### 核心特性

1. **纯字符串别名**：aliases字段存储多个别名名称，无需对应ModelList记录
2. **同级路由扩展**：别名不是降级，而是同级别扩展，一次性使用所有名称搜索渠道
3. **无数量限制**：每个模型可配置任意数量的别名
4. **灵活命名**：别名可以是任意字符串，用于匹配更多渠道配置

## 工作原理

### 1. 路由流程（重要修订）

**正确流程**：
```
入参（模型名称）
   ↓
Key 映射（API Key model_mappings）
   ↓
别名扩展（获取所有关联名称）
   ↓
搜索渠道（使用所有名称一起搜索）
   ↓
过滤与负载均衡
```

**关键点**：
- 别名是**同级扩展**，不是降级
- 使用所有名称（模型名 + 别名）一次性搜索渠道
- 最大化可用渠道池

**代码实现**（[ChannelRouterService.php](laravel/app/Services/Router/ChannelRouterService.php)）：
```php
public function selectChannel(string $model, array $context = []): Channel
{
    // 获取所有关联的模型名称（模型名 + 别名）
    $modelNames = $this->getModelNamesWithAliases($model);

    // 使用所有名称搜索渠道
    $channels = $this->getAvailableChannelsForModels($modelNames);

    // ... 后续过滤和负载均衡
}

protected function getModelNamesWithAliases(string $model): array
{
    return ModelService::resolveModelWithAliases($model);
}

protected function getAvailableChannelsForModels(array $modelNames): Collection
{
    $channelIds = [];
    foreach ($modelNames as $modelName) {
        $ids = $this->getChannelIdsForModel($modelName);
        $channelIds = array_merge($channelIds, $ids);
    }

    $channelIds = array_unique($channelIds);

    return Channel::whereIn('id', $channelIds)
        ->where('status', 'active')
        ->where('status2', 'normal')
        ->get();
}
```

### 2. 别名存储与查询

**数据存储**：
- aliases字段存储为JSON数组
- 别名可以是任意字符串，无需对应ModelList记录
- 无自动同步行为

**示例数据**：
```json
{
  "model_name": "glm-4",
  "aliases": ["glm-4-plus", "glm-4-turbo", "glm-4-flash"]
}
```

**查询逻辑**：
- `findModelByAnyName('glm-4-plus')` → 找到"glm-4"模型
- `getAllNames()` → 返回["glm-4", "glm-4-plus", "glm-4-turbo", "glm-4-flash"]
- 用于路由时搜索所有支持这些名称的渠道

## 使用示例

### 配置场景

假设系统中：
- 模型"glm-4"，配置别名["glm-4-plus", "glm-4-turbo"]
- 渠道A的channelModels配置model_name为"glm-4"
- 渠道B的channelModels配置model_name为"glm-4-plus"
- 渠道C的channelModels配置model_name为"glm-4-turbo"

### 配置别名

在管理后台为 `glm-4` 配置别名 `glm-4-plus` 和 `glm-4-turbo`。

### 路由效果

**请求 glm-4**：
1. 获取所有名称：`['glm-4', 'glm-4-plus', 'glm-4-turbo']`
2. 搜索渠道：找到A、B、C三个渠道
3. 应用过滤（API Key、协议、User-Agent）
4. 负载均衡选择

**请求 glm-4-plus**：
1. 通过别名找到"glm-4"模型
2. 获取所有名称：`['glm-4', 'glm-4-plus', 'glm-4-turbo']`
3. 搜索渠道：找到A、B、C三个渠道

**优势**：
- 获得更多可用渠道
- 提升请求成功率
- 避免不必要的错误

## 技术细节

### 性能优化

1. **单次查询**：所有别名对应的渠道一次性获取
2. **缓存机制**：别名查询结果缓存30秒
3. **去重优化**：使用 array_unique 避免重复渠道
