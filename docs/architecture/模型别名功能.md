# 模型别名功能设计文档（修订版）

## 功能概述

模型别名功能允许为 AI 模型配置多个关联名称，主要用于**渠道路由扩展**场景。当请求某个模型时，系统自动使用该模型的所有别名名称一起搜索渠道，最大化渠道可用性。

### 核心特性

1. **多向对称关联**：所有关联的模型完全等价，配置任一模型的别名都会自动同步到其他模型
2. **同级路由扩展**：别名不是降级，而是同级别扩展，一次性使用所有名称搜索渠道
3. **无数量限制**：每个模型可配置任意数量的别名
4. **数据一致性保证**：别名修改自动同步，保证双向关系一致

## 工作原理

### 1. 路由流程（重要修订）

**正确流程**：
```
入参（模型名称）
   ↓
Key 映射（API Key model_mappings）
   ↓
别名扩展（获取所有关联名称）
   ↓
搜索渠道（使用所有名称一起搜索）
   ↓
过滤与负载均衡
```

**关键点**：
- 别名是**同级扩展**，不是降级
- 使用所有名称（模型名 + 别名）一次性搜索渠道
- 最大化可用渠道池

**代码实现**（[ChannelRouterService.php](laravel/app/Services/Router/ChannelRouterService.php)）：
```php
public function selectChannel(string $model, array $context = []): Channel
{
    // 获取所有关联的模型名称（模型名 + 别名）
    $modelNames = $this->getModelNamesWithAliases($model);

    // 使用所有名称搜索渠道
    $channels = $this->getAvailableChannelsForModels($modelNames);

    // ... 后续过滤和负载均衡
}

protected function getModelNamesWithAliases(string $model): array
{
    return ModelService::resolveModelWithAliases($model);
}

protected function getAvailableChannelsForModels(array $modelNames): Collection
{
    $channelIds = [];
    foreach ($modelNames as $modelName) {
        $ids = $this->getChannelIdsForModel($modelName);
        $channelIds = array_merge($channelIds, $ids);
    }

    $channelIds = array_unique($channelIds);

    return Channel::whereIn('id', $channelIds)
        ->where('status', 'active')
        ->where('status2', 'normal')
        ->get();
}
```

## 使用示例

### 配置场景

假设系统中：
- `glm-5`：部分渠道支持（channel_id: 2, 5, 7）
- `GLM-5`：其他渠道支持（channel_id: 3）

### 配置别名

在管理后台为 `glm-5` 配置别名 `GLM-5`，系统自动双向同步。

### 路由效果

**请求 glm-5**：
1. 获取所有名称：`['glm-5', 'GLM-5']`
2. 搜索渠道：channel_id: [2, 5, 7] + [3] = [2, 3, 5, 7]
3. 应用过滤（API Key、协议、User-Agent）
4. 负载均衡选择

**优势**：
- 获得更多可用渠道
- 提升请求成功率
- 避免不必要的错误

## 技术细节

### 性能优化

1. **单次查询**：所有别名对应的渠道一次性获取
2. **缓存机制**：别名查询结果缓存30秒
3. **去重优化**：使用 array_unique 避免重复渠道
