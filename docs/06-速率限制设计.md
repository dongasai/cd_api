## 三、限流算法

### 3.1 滑动窗口算法

```
┌─────────────────────────────────────────────────────────────────────┐
│                        滑动窗口算法                                  │
└─────────────────────────────────────────────────────────────────────┘

原理:
使用 Redis Sorted Set (ZSET) 存储请求时间戳，实现精确的滑动窗口

Redis 实现:
Key: cdapi:rate:{identifier}
Type: ZSET
Member: 请求时间戳 (毫秒)
Score: 请求时间戳 (毫秒)

算法步骤:
1. 当前时间戳: now = 1709520000000 (毫秒)
2. 窗口大小: window = 60000 (60秒)
3. 窗口开始: start = now - window = 1709519940000

4. 移除过期记录:
   ZREMRANGEBYSCORE key 0 start

5. 统计当前窗口内请求数:
   count = ZCARD key

6. 判断是否超限:
   if count >= limit:
       return "Rate limit exceeded"
   
7. 记录本次请求:
   ZADD key now now

8. 设置过期时间:
   EXPIRE key window

Lua 脚本实现:
local key = KEYS[1]
local limit = tonumber(ARGV[1])
local window = tonumber(ARGV[2])
local now = tonumber(ARGV[3])

redis.call('ZREMRANGEBYSCORE', key, 0, now - window)
local count = redis.call('ZCARD', key)

if count >= limit then
    return {0, count, limit}
end

redis.call('ZADD', key, now, now)
redis.call('EXPIRE', key, window / 1000)

return {1, count + 1, limit}
```

### 3.2 令牌桶算法

```
┌─────────────────────────────────────────────────────────────────────┐
│                        令牌桶算法                                    │
└─────────────────────────────────────────────────────────────────────┘

原理:
以固定速率向桶中添加令牌，请求消耗令牌，桶满则丢弃新令牌

参数:
├─── capacity: 桶容量 (最大令牌数)
├─── rate: 令牌生成速率 (令牌/秒)
└─── tokens: 当前令牌数

Redis 实现:
Key: cdapi:token_bucket:{identifier}
Fields:
  - tokens: 当前令牌数
  - last_refill: 上次填充时间

Lua 脚本:
local key = KEYS[1]
local capacity = tonumber(ARGV[1])
local rate = tonumber(ARGV[2])
local requested = tonumber(ARGV[3])
local now = tonumber(ARGV[4])

local info = redis.call('HMGET', key, 'tokens', 'last_refill')
local tokens = tonumber(info[1]) or capacity
local last_refill = tonumber(info[2]) or now

-- 计算新增令牌
local elapsed = now - last_refill
local new_tokens = math.floor(elapsed * rate / 1000)
tokens = math.min(capacity, tokens + new_tokens)

-- 检查是否有足够令牌
if tokens < requested then
    redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
    return {0, tokens, capacity}
end

-- 消耗令牌
tokens = tokens - requested
redis.call('HMSET', key, 'tokens', tokens, 'last_refill', now)
redis.call('EXPIRE', key, 3600)

return {1, tokens, capacity}
```

### 3.3 漏桶算法

```
┌─────────────────────────────────────────────────────────────────────┐
│                        漏桶算法                                      │
└─────────────────────────────────────────────────────────────────────┘

原理:
请求进入队列，以固定速率处理，队列满则丢弃请求

参数:
├─── capacity: 桶容量 (队列大小)
├─── rate: 处理速率 (请求/秒)
└─── water: 当前水位

适用场景:
├─── 需要平滑流量的场景
├─── 下游服务处理能力有限
└─── 需要削峰填谷
```

### 3.4 算法对比

```
┌─────────────────────────────────────────────────────────────────────┐
│                        算法对比                                      │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────┬─────────────┬─────────────┬─────────────┬───────────┐
│ 算法        │ 精确度      │ 内存消耗    │ 允许突发    │ 适用场景  │
├─────────────┼─────────────┼─────────────┼─────────────┼───────────┤
│ 滑动窗口    │ 高          │ 中          │ 否          │ 精确限流  │
│ 令牌桶      │ 中          │ 低          │ 是          │ 允许突发  │
│ 漏桶        │ 中          │ 低          │ 否          │ 平滑流量  │
└─────────────┴─────────────┴─────────────┴─────────────┴───────────┘

推荐选择:
├─── API 请求限流: 滑动窗口 (精确控制)
├─── Token 限流: 令牌桶 (允许突发)
└─── 下游保护: 漏桶 (平滑流量)
```

### 4.2 重试策略

```
┌─────────────────────────────────────────────────────────────────────┐
│                        客户端重试策略                                │
└─────────────────────────────────────────────────────────────────────┘

指数退避重试:
├─── 第 1 次重试: 等待 1 秒
├─── 第 2 次重试: 等待 2 秒
├─── 第 3 次重试: 等待 4 秒
├─── 第 4 次重试: 等待 8 秒
└─── 最大等待: 60 秒

重试建议:
1. 检查 Retry-After Header
2. 使用指数退避
3. 添加随机抖动 (避免惊群效应)
4. 设置最大重试次数

伪代码:
function retryWithBackoff(maxRetries = 5):
    for i in range(maxRetries):
        response = makeRequest()
        if response.status != 429:
            return response
        
        waitTime = min(2^i + random(0, 1), 60)
        if response.headers['Retry-After']:
            waitTime = max(waitTime, response.headers['Retry-After'])
        
        sleep(waitTime)
    
    return error
```

## 七、实现细节

### 7.1 中间件实现

```
┌─────────────────────────────────────────────────────────────────────┐
│                     RateLimiter 中间件                              │
└─────────────────────────────────────────────────────────────────────┘

处理流程:
1. 获取限流配置
   ├─── 全局配置
   ├─── 租户配置
   └─── 令牌配置

2. 构建限流 Key
   ├─── 全局: cdapi:rate:global
   ├─── 租户: cdapi:rate:tenant:{tenant_id}
   └─── 令牌: cdapi:rate:token:{token_id}

3. 执行限流检查
   ├─── 请求次数检查
   ├─── Token 数检查
   └─── 并发检查

4. 处理检查结果
   ├─── 通过: 继续处理请求
   └─── 拒绝: 返回 429 错误

5. 设置响应头
   ├─── X-RateLimit-Limit
   ├─── X-RateLimit-Remaining
   └─── X-RateLimit-Reset

6. 请求完成后更新计数
   └─── 使用实际 Token 数更新
```

### 7.2 性能优化

```
┌─────────────────────────────────────────────────────────────────────┐
│                        限流性能优化                                  │
└─────────────────────────────────────────────────────────────────────┘

Redis 优化:
├─── 使用 Lua 脚本保证原子性
├─── 使用 Pipeline 批量执行
├─── 使用本地缓存减少 Redis 访问
└─── 使用 Redis Cluster 分散压力

本地缓存:
├─── 缓存限流配置
├─── 缓存限流计数 (短期)
└─── 定期同步到 Redis

批量检查:
├─── 一次检查多个维度
├─── 使用 Pipeline 减少网络往返
└─── 并行检查多个 Key
```
