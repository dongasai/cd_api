# 认证授权设计

## 一、认证架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          认证流程架构                                │
└─────────────────────────────────────────────────────────────────────┘

                        ┌─────────────────┐
                        │   客户端请求     │
                        │ Authorization:  │
                        │ Bearer sk-xxx   │
                        └────────┬────────┘
                                 │
                                 ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     AuthenticateToken 中间件                         │
│                                                                     │
│  1. 提取令牌                                                        │
│     └─── 从 Authorization Header 提取 Bearer Token                  │
│                                                                     │
│  2. 解析前缀                                                        │
│     └─── sk-xxx → prefix = "sk"                                    │
│                                                                     │
│  3. 查询缓存                                                        │
│     └─── Key: cdapi:token:validate:sk:{hash_prefix}                │
│                                                                     │
│  4. 缓存未命中 → 查询数据库                                         │
│     └─── SELECT * FROM tokens WHERE key_prefix = 'sk'              │
│         AND status = 'active'                                       │
│                                                                     │
│  5. 验证令牌哈希                                                    │
│     └─── bcrypt_verify(token, key_hash)                            │
│                                                                     │
│  6. 检查过期时间                                                    │
│     └─── expires_at IS NULL OR expires_at > NOW()                  │
│                                                                     │
│  7. 加载租户信息                                                    │
│     └─── 查询租户并验证状态                                         │
│                                                                     │
│  8. 设置上下文                                                      │
│     └─── tenant(), token() 全局辅助函数                             │
│                                                                     │
│  9. 缓存验证结果                                                    │
│     └─── TTL: 60 秒                                                 │
└─────────────────────────────────────────────────────────────────────┘
                                 │
                                 ▼
                        ┌─────────────────┐
                        │   控制器处理     │
                        └─────────────────┘
```

## 二、令牌设计

### 2.1 令牌格式

```
┌─────────────────────────────────────────────────────────────────────┐
│                          令牌格式规范                                │
└─────────────────────────────────────────────────────────────────────┘

格式: {prefix}_{random_string}

示例:
┌─────────────────────┬──────────────────────────────────────────────┐
│ 令牌                │ 说明                                         │
├─────────────────────┼──────────────────────────────────────────────┤
│ sk-abc123def456...  │ 标准令牌                                     │
│ sk_live_xyz789...   │ 生产环境令牌                                 │
│ sk_test_abc123...   │ 测试环境令牌                                 │
│ pk_prod_def456...   │ 公开令牌 (只读权限)                          │
│ admin_secret_123... │ 管理员令牌                                   │
└─────────────────────┴──────────────────────────────────────────────┘

令牌组成:
┌─────────────┬──────────────────────────────────────────────────────┐
│ 前缀        │ 随机字符串 (32-64 字符)                              │
│ sk_live_    │ abc123def456ghi789jkl012mno345pqr678stu901vwx...    │
│ (可配置)    │ (加密安全的随机字节，Base62 编码)                    │
└─────────────┴──────────────────────────────────────────────────────┘

前缀规则:
- sk: 标准密钥
- pk: 公开密钥 (受限权限)
- {env}_: 环境标识 (live, test, dev)
- {role}_: 角色标识 (admin, user, service)
```

### 2.2 令牌存储

```
┌─────────────────────────────────────────────────────────────────────┐
│                          令牌存储策略                                │
└─────────────────────────────────────────────────────────────────────┘

数据库存储:
┌─────────────────┬──────────────────────────────────────────────────┐
│ 字段            │ 存储内容                                         │
├─────────────────┼──────────────────────────────────────────────────┤
│ key_prefix      │ "sk" (用于快速查询和索引)                        │
│ key_hash        │ bcrypt(token, cost=12) (不可逆哈希)              │
│ name            │ "Production API Key" (友好名称)                  │
└─────────────────┴──────────────────────────────────────────────────┘

安全措施:
1. 原始令牌只在创建时显示一次
2. 使用 bcrypt 哈希存储 (cost=12)
3. 前缀用于快速定位，减少哈希验证次数
4. 支持令牌撤销和过期

生成流程:
1. 生成 32 字节加密安全随机数
2. Base62 编码得到随机字符串
3. 添加前缀: sk_{random_string}
4. 计算 bcrypt 哈希
5. 存储前缀和哈希
6. 返回原始令牌 (仅此一次)
```

### 2.3 令牌验证流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        令牌验证详细流程                              │
└─────────────────────────────────────────────────────────────────────┘

输入: Authorization: Bearer sk-abc123def456...

Step 1: 提取令牌
├─── 验证 Header 格式
├─── 提取 Bearer Token
└─── 令牌: sk-abc123def456...

Step 2: 解析前缀
├─── 分割令牌: prefix = "sk", body = "abc123def456..."
└─── 验证前缀有效性

Step 3: 查询缓存
├─── Key: cdapi:token:validate:sk:{sha256(body)[:8]}
├─── 缓存命中 → 直接返回验证结果
└─── 缓存未命中 → 继续数据库查询

Step 4: 数据库查询
├─── SELECT * FROM tokens
│    WHERE key_prefix = 'sk'
│    AND status = 'active'
├─── 可能返回多条记录 (同前缀的令牌)
└─── 需要逐一验证哈希

Step 5: 哈希验证
├─── 遍历查询结果
├─── bcrypt_verify(token, key_hash)
└─── 找到匹配的令牌记录

Step 6: 状态验证
├─── 检查令牌状态: status = 'active'
├─── 检查过期时间: expires_at IS NULL OR expires_at > NOW()
└─── 检查最后使用时间

Step 7: 租户验证
├─── 加载关联的租户
├─── 检查租户状态: status = 'active'
└─── 检查租户配额

Step 8: 权限加载
├─── 加载令牌权限配置
├─── 加载允许的模型列表
└─── 加载速率限制配置

Step 9: 设置上下文
├─── 设置当前租户: tenant()
├─── 设置当前令牌: token()
└─── 设置权限上下文

Step 10: 缓存结果
├─── 缓存验证结果 (TTL: 60s)
└─── 包含: tenant_id, token_id, permissions

Step 11: 更新最后使用时间
├─── 异步更新 last_used_at
└─── 避免阻塞请求
```

## 三、权限模型

### 3.1 RBAC 模型

```
┌─────────────────────────────────────────────────────────────────────┐
│                          权限模型架构                                │
└─────────────────────────────────────────────────────────────────────┘

租户
  │
  ├─── 租户级别配置
  │    ├── allowed_models: ["gpt-4", "claude-3-opus"]
  │    ├── default_rate_limit: {...}
  │    └── features: {...}
  │
  └─── 令牌
       │
       ├─── 令牌级别配置
       │    ├── permissions: {...}
       │    ├── allowed_models: [...] (覆盖租户配置)
       │    └── rate_limit: {...} (覆盖租户配置)
       │
       └─── 权限继承规则
            ├─── 令牌配置优先于租户配置
            ├─── 令牌未配置则继承租户配置
            └─── 最终取交集 (更严格的限制)
```

### 3.2 权限配置结构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        权限配置 JSON 结构                            │
└─────────────────────────────────────────────────────────────────────┘

租户配置:
{
    "allowed_models": ["gpt-4", "gpt-3.5-turbo", "claude-3-opus"],
    "default_rate_limit": {
        "requests_per_minute": 60,
        "tokens_per_minute": 10000,
        "requests_per_day": 10000
    },
    "features": {
        "streaming": true,
        "embeddings": true,
        "function_calling": true
    },
    "channel_restrictions": {
        "allowed_channels": [1, 2, 3],
        "blocked_channels": []
    }
}

令牌配置:
{
    "permissions": {
        "endpoints": ["chat", "completions", "embeddings"],
        "max_tokens_per_request": 4000,
        "max_requests_per_day": 1000
    },
    "allowed_models": ["gpt-4"],  // 只允许 gpt-4，覆盖租户配置
    "rate_limit": {
        "requests_per_minute": 30,  // 更严格的限制
        "tokens_per_minute": 5000
    }
}
```

### 3.3 权限检查流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                          权限检查流程                                │
└─────────────────────────────────────────────────────────────────────┘

检查模型访问权限:
┌─────────────────────────────────────────────────────────────────────┐
│ checkModelAccess(model: string): bool                               │
├─────────────────────────────────────────────────────────────────────┤
│ 1. 获取令牌的 allowed_models                                        │

│                                                                     │
│ 2. 检查请求的 token 数是否超限                                      │

│    └─── 如果请求中没有指定，使用估算值                              │
│                                                                     │
│ 3. 返回结果                                                         │
└─────────────────────────────────────────────────────────────────────┘
```

## 四、多租户设计

### 4.1 租户隔离

```
┌─────────────────────────────────────────────────────────────────────┐
│                          租户隔离策略                                │
└─────────────────────────────────────────────────────────────────────┘

数据隔离:
├─── 审计日志按 tenant_id 隔离
├─── 令牌按 tenant_id 隔离
├─── 配额按租户独立计算
└─── 统计数据按租户聚合

配置隔离:
├─── 每个租户独立的模型权限配置
├─── 每个租户独立的速率限制配置
├─── 每个租户独立的渠道限制
└─── 每个租户独立的功能开关

运行时隔离:
├─── 请求上下文绑定当前租户
├─── 缓存 Key 包含 tenant_id
├─── 日志记录包含 tenant_id
└─── 错误追踪关联 tenant_id
```

### 4.2 租户解析

```
┌─────────────────────────────────────────────────────────────────────┐
│                          租户解析策略                                │
└─────────────────────────────────────────────────────────────────────┘

方式一: 通过令牌关联 (推荐)
├─── 令牌创建时关联 tenant_id
├─── 验证令牌时自动解析租户
└─── 无需额外参数

方式二: 通过 Header 指定
├─── X-Tenant-Slug: acme
├─── 适用于多租户共享令牌场景
└─── 需要令牌有跨租户权限

方式三: 通过子域名
├─── acme.api.example.com
├─── 从 Host Header 解析租户
└─── 适用于独立部署场景

解析优先级:
1. 令牌关联的租户 (最高优先级)
2. X-Tenant-Slug Header
3. 子域名解析
```

### 4.3 租户上下文管理

```
┌─────────────────────────────────────────────────────────────────────┐
│                        租户上下文管理                                │
└─────────────────────────────────────────────────────────────────────┘

全局辅助函数:
function tenant(): ?Tenant
function token(): ?Token
function tenant_id(): int

中间件设置:
class ResolveTenant
{
    public function handle($request, $next)
    {
        // 从令牌获取租户
        $token = $this->getToken($request);
        $tenant = $token->tenant;
        
        // 验证租户状态
        if ($tenant->status !== 'active') {
            throw new TenantSuspendedException();
        }
        
        // 设置上下文
        app()->instance('current_tenant', $tenant);
        app()->instance('current_token', $token);
        
        return $next($request);
    }
}

使用示例:
$tenant = tenant();
$token = token();
$tenantId = tenant_id();

// 在模型中自动添加租户条件
class AuditLog extends Model
{
    protected static function booted()
    {
        static::addGlobalScope('tenant', function ($query) {
            if ($tenantId = tenant_id()) {
                $query->where('tenant_id', $tenantId);
            }
        });
    }
}
```

## 五、配额管理

### 5.1 配额类型

```
┌─────────────────────────────────────────────────────────────────────┐
│                          配额类型设计                                │
└─────────────────────────────────────────────────────────────────────┘

金额配额:
├─── quota_limit: 1000.00 USD
├─── quota_used: 250.50 USD
├─── quota_reset_at: 每月重置
└─── 超额后拒绝请求 (HTTP 402)

请求次数配额:
├─── requests_per_day: 10000
├─── requests_per_month: 100000
└─── 超额后拒绝请求 (HTTP 429)

Token 配额:
├─── tokens_per_day: 1000000
├─── tokens_per_month: 10000000
└─── 超额后拒绝请求 (HTTP 429)

组合配额:
├─── 可同时配置多种配额
├─── 任一配额超限即拒绝
└─── 优先返回最严格的限制信息
```

### 5.2 配额检查流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                          配额检查流程                                │
└─────────────────────────────────────────────────────────────────────┘

请求到达
    │
    ▼
检查金额配额
    │
    ├─── quota_limit IS NOT NULL
    │    │
    │    ├─── quota_used >= quota_limit
    │    │    └─── 抛出 QuotaExceededException
    │    │
    │    └─── 通过
    │
    └─── quota_limit IS NULL
         └─── 跳过金额配额检查
    │
    ▼
检查请求次数配额
    │
    ├─── 从 Redis 获取今日请求数
    │    Key: cdapi:quota:{tenant_id}:requests:{date}
    │
    ├─── 请求数 >= limit
    │    └─── 抛出 RateLimitException
    │
    └─── 通过
    │
    ▼
检查 Token 配额
    │
    ├─── 从 Redis 获取今日 Token 数
    │    Key: cdapi:quota:{tenant_id}:tokens:{date}
    │
    ├─── 估算请求 Token 数
    │    └─── 输入 Token + max_tokens
    │
    ├─── 预估后超限
    │    └─── 抛出 RateLimitException
    │
    └─── 通过
    │
    ▼
继续处理请求
    │
    ▼
请求完成后更新配额
    │
    ├─── 更新金额配额 (异步)
    │    └─── UPDATE tenants SET quota_used = quota_used + :cost
    │
    ├─── 更新请求次数 (Redis INCR)
    │
    └─── 更新 Token 数 (Redis INCRBY)
```

### 5.3 配额重置

```
┌─────────────────────────────────────────────────────────────────────┐
│                          配额重置策略                                │
└─────────────────────────────────────────────────────────────────────┘

金额配额重置:
├─── 每月 1 日 00:00 UTC 自动重置
├─── quota_used = 0
├─── quota_reset_at = 下月 1 日
└─── 通过定时任务执行

请求次数重置:
├─── 每日 00:00 UTC 自动重置
├─── Redis Key 自动过期
└─── 无需手动干预

Token 配额重置:
├─── 每日 00:00 UTC 自动重置
├─── Redis Key 自动过期
└─── 无需手动干预

手动重置:
├─── 管理后台支持手动重置
├─── API 支持管理员重置
└─── 记录重置操作日志
```

## 六、安全措施

### 6.1 令牌安全

```
┌─────────────────────────────────────────────────────────────────────┐
│                          令牌安全措施                                │
└─────────────────────────────────────────────────────────────────────┘

存储安全:
├─── 使用 bcrypt 哈希存储 (cost=12)
├─── 原始令牌只在创建时显示一次
├─── 数据库字段加密存储 API Key
└─── 敏感操作记录审计日志

传输安全:
├─── 强制 HTTPS
├─── 令牌不记录在 URL 中
├─── 令牌不记录在日志中
└─── 响应中不返回完整令牌

使用安全:
├─── 支持令牌过期时间
├─── 支持令牌撤销
├─── 支持令牌使用频率限制
└─── 异常使用触发告警
```

### 6.2 防护措施

```
┌─────────────────────────────────────────────────────────────────────┐
│                          安全防护措施                                │
└─────────────────────────────────────────────────────────────────────┘

暴力破解防护:
├─── 令牌验证失败计数
├─── 连续失败 N 次后临时封禁 IP
├─── 封禁时间递增 (1分钟 → 5分钟 → 1小时)
└─── 记录可疑行为

令牌泄露检测:
├─── 检测异常使用模式
├─── 检测多地登录
├─── 检测异常请求量
└─── 自动告警和临时冻结

请求验证:
├─── 验证请求来源 IP
├─── 验证 User-Agent
├─── 验证请求频率
└─── 验证请求大小
```
