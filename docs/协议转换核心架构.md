# 协议转换核心架构

## 架构概览

CdApi 的协议转换系统采用 **中间层 DTO 模式**，通过统一的 Shared\DTO 作为协议转换枢纽。

```
客户端协议 ──→ ProtocolRequest ──→ Shared\DTO（无状态） ──→ 目标协议 ProtocolRequest ──→ 上游API
上游响应    ──→ ProtocolResponse ──→ Shared\DTO（无状态） ──→ 客户端协议 ProtocolResponse
```

## 核心组件

### 1. ProtocolConverter（协议转换器）

`laravel/app/Services/Protocol/ProtocolConverter.php`

**职责**：协议结构体之间的转换协调
- `convertRequest()` - 请求结构体转换
- `convertResponse()` - 响应结构体转换
- `convertStream()` - 流式事件转换
- `normalizeRequest()` - 原始请求解析
- `denormalizeResponse()` - 响应构建

### 2. DriverManager（驱动管理器）

`laravel/app/Services/Protocol/DriverManager.php`

**职责**：协议驱动的注册、解析、缓存
- `register()` - 注册驱动
- `driver()` - 获取驱动实例（延迟解析）
- `hasDriver()` - 检查协议支持
- `getRegisteredDrivers()` - 获取支持的协议列表

### 3. DriverInterface（驱动接口）

`laravel/app/Services/Protocol/Driver/DriverInterface.php`

**核心方法**：
| 方法 | 职责 |
|------|------|
| `parseRequest()` | 原始请求 → ProtocolRequest |
| `buildResponse()` | ProtocolResponse → 响应数组 |
| `buildStreamChunk()` | StreamChunk → SSE 数据块 |
| `buildStreamDone()` | 流式结束标记 |
| `extractModel()` | 提取请求中的模型名 |

### 4. ProtocolRequest/ProtocolResponse（协议接口）

`laravel/app/Services/Protocol/Contracts/`

**双向转换契约**：
- `toSharedDTO()` - 转为统一 DTO
- `fromSharedDTO()` - 从统一 DTO 创建
- `toArray()` / `fromArray()` - 数组序列化

### 5. Shared\DTO（统一数据层）

`laravel/app/Services/Shared/DTO/`

| DTO | 用途 |
|-----|------|
| `Request` | 统一请求结构（model, messages, tools 等） |
| `Response` | 统一响应结构（id, choices, usage 等） |
| `StreamChunk` | 统一流式块（contentDelta, finishReason 等） |

## 转换流程

### 请求转换

```php
// 1. 解析原始请求
$request = $converter->normalizeRequest($rawData, 'openai_chat_completions');

// 2. 转换为目标协议（自动判断是否需要转换）
$targetRequest = $converter->convertRequest($request, 'anthropic_messages');

// 3. 获取转换后的数组发送上游
$upstreamData = $targetRequest->toArray();
```

### 响应转换

```php
// 1. 从上游响应创建源协议响应
$response = MessagesResponse::fromArray($upstreamData);

// 2. 转换为客户端协议格式
$clientResponse = $converter->convertResponse($response, 'openai_chat_completions');

// 3. 构建输出
$output = $converter->denormalizeResponse($clientResponse, 'openai_chat_completions');
```

### 流式转换

```php
foreach ($converter->convertStream($stream, 'openai_chat_completions') as $chunk) {
    echo $chunk; // SSE 格式输出
}
```

## 已支持协议

| 协议名 | 驱动类 | 备注 |
|--------|--------|------|
| `openai_chat_completions` | OpenAiChatCompletionsDriver | OpenAI Chat API |
| `anthropic_messages` | AnthropicMessagesDriver | Anthropic Messages API |

**别名映射**：`openai` → `openai_chat_completions`，`anthropic` → `anthropic_messages`

## 设计原则

1. **无状态转换**：ProtocolRequest/ProtocolResponse 是纯数据容器，不含业务逻辑
2. **中间层隔离**：Shared\DTO 作为协议无关的中间层，解耦协议差异
3. **驱动可扩展**：实现 DriverInterface 即可注册新协议
4. **同协议优化**：源/目标协议相同时直接返回，跳过转换
