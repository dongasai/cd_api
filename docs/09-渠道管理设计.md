# 渠道管理设计

## 一、渠道概述

```
┌─────────────────────────────────────────────────────────────────────┐
│                          渠道管理架构                                │
└─────────────────────────────────────────────────────────────────────┘

渠道 (Channel) 是连接上游 AI 服务提供商的配置单元，负责：
├─── 定义 API 端点和认证信息
├─── 配置支持的模型列表
├─── 设置负载均衡权重和优先级
├─── 管理运营状态和故障切换
└─── 支持继承和参数覆盖

渠道类型:
┌─────────────────┬──────────────────────────────────────────────────┐
│ Provider        │ 说明                                             │
├─────────────────┼──────────────────────────────────────────────────┤
│ openai          │ OpenAI 官方 API                                  │
│ azure           │ Azure OpenAI Service                             │
│ anthropic       │ Anthropic Claude API                             │
│ google          │ Google Gemini API                                │
│ custom          │ 自定义 OpenAI 兼容 API                            │
└─────────────────┴──────────────────────────────────────────────────┘
```

## 二、渠道继承设计

### 2.1 继承模型

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道继承模型                                  │
└─────────────────────────────────────────────────────────────────────┘

继承关系:
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│                      ┌──────────────┐                              │
│                      │  基础渠道     │                              │
│                      │  (openai)    │                              │
│                      │  base_url    │                              │
│                      │  api_key     │                              │
│                      │  models      │                              │
│                      │  config      │                              │
│                      └──────┬───────┘                              │
│                             │                                       │
│              ┌──────────────┼──────────────┐                       │
│              │              │              │                       │
│              ▼              ▼              ▼                       │
│      ┌──────────────┐┌──────────────┐┌──────────────┐             │
│      │ 子渠道 A     ││ 子渠道 B     ││ 子渠道 C     │             │
│      │ (gpt-4 专用) ││ (gpt-3.5专用)││ (继承+覆盖)  │             │
│      │ models:      ││ models:      ││ weight: 5    │             │
│      │  [gpt-4]     ││  [gpt-3.5]   ││ priority: 1  │             │
│      │ weight: 3    ││  weight: 2   ││              │             │
│      └──────────────┘└──────────────┘└──────────────┘             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

继承规则:
┌─────────────────┬──────────────────────────────────────────────────┐
│ 规则            │ 说明                                             │
├─────────────────┼──────────────────────────────────────────────────┤
│ 可继承字段      │ base_url, api_key, models, config, timeout 等   │
│ 不可继承字段    │ id, name, parent_id, status                     │
│ 覆盖规则        │ 子渠道字段优先于父渠道                          │
│ 多级继承        │ 支持多级继承 (最多 5 级)                        │
│ 循环检测        │ 禁止循环继承                                    │
└─────────────────┴──────────────────────────────────────────────────┘
```

### 2.2 数据库设计

渠道表及相关表的详细设计请参考 [数据库设计文档](./02-数据库设计.md)：

- **channels** - 渠道表（支持继承机制）
- **channel_groups** - 渠道分组表
- **channel_group_pivot** - 渠道分组关联表
- **channel_tags** - 渠道标签表
- **channel_tag_pivot** - 渠道标签关联表

**核心继承字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| parent_id | BIGINT | 父渠道 ID，NULL 表示根渠道 |
| slug | VARCHAR(100) | 渠道唯一标识，用于 API 引用 |
| inherit_mode | ENUM | 继承模式: merge(合并), override(覆盖), extend(扩展) |

### 2.3 继承模式

```
┌─────────────────────────────────────────────────────────────────────┐
│                        继承模式详解                                  │
└─────────────────────────────────────────────────────────────────────┘

模式一: merge (合并) - 默认
├─── 数组类型: 合并父子的数组 (去重)
├─── 对象类型: 深度合并，子覆盖父
├─── 标量类型: 子覆盖父
└─── 适用场景: 扩展父渠道配置

示例:
父渠道 models: ["gpt-4", "gpt-3.5-turbo"]
子渠道 models: ["gpt-4-turbo"]
合并结果 models: ["gpt-4", "gpt-3.5-turbo", "gpt-4-turbo"]

模式二: override (覆盖)
├─── 完全使用子渠道的值
├─── 忽略父渠道的对应字段
└─── 适用场景: 完全自定义配置

示例:
父渠道 models: ["gpt-4", "gpt-3.5-turbo"]
子渠道 models: ["gpt-4-turbo"]
覆盖结果 models: ["gpt-4-turbo"]

模式三: extend (扩展)
├─── 数组类型: 子数组追加到父数组
├─── 对象类型: 子对象合并到父对象
├─── 标量类型: 子覆盖父
└─── 适用场景: 保留父配置并添加新配置

示例:
父渠道 config: {"timeout": 120, "retry": 3}
子渠道 config: {"cache": true}
扩展结果 config: {"timeout": 120, "retry": 3, "cache": true}
```

### 2.4 继承解析流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                      渠道继承解析流程                                │
└─────────────────────────────────────────────────────────────────────┘

输入: channel_id = 5

Step 1: 加载渠道链
┌─────────────────────────────────────────────────────────────────────┐
│ SELECT * FROM channels WHERE id IN (                               │
│   WITH RECURSIVE channel_tree AS (                                 │
│     SELECT id, parent_id, 1 AS level FROM channels WHERE id = 5    │
│     UNION ALL                                                      │
│     SELECT c.id, c.parent_id, ct.level + 1                         │
│     FROM channels c                                                 │
│     JOIN channel_tree ct ON c.id = ct.parent_id                    │
│     WHERE ct.level < 5  -- 限制最大层级                            │
│   )                                                                │
│   SELECT id FROM channel_tree ORDER BY level DESC                  │
│ )                                                                  │
└─────────────────────────────────────────────────────────────────────┘

结果: [Channel(1), Channel(3), Channel(5)]
      根渠道 → 中间渠道 → 目标渠道

Step 2: 按层级合并配置
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│ Channel(1) 根渠道:                                                  │
│ {                                                                   │
│   base_url: "https://api.openai.com",                              │
│   api_key: "sk-xxx",                                               │
│   models: ["gpt-4", "gpt-3.5-turbo"],                              │
│   config: {timeout: 120, retry: 3}                                 │
│ }                                                                   │
│                                                                     │
│ Channel(3) 中间渠道 (inherit_mode: merge):                          │
│ {                                                                   │
│   models: ["gpt-4-turbo"],                                         │
│   weight: 3                                                        │
│ }                                                                   │
│                                                                     │
│ 合并结果:                                                           │
│ {                                                                   │
│   base_url: "https://api.openai.com",  // 继承                     │
│   api_key: "sk-xxx",                    // 继承                     │
│   models: ["gpt-4", "gpt-3.5-turbo", "gpt-4-turbo"],  // 合并      │
│   config: {timeout: 120, retry: 3},     // 继承                     │
│   weight: 3                             // 子渠道定义               │
│ }                                                                   │
│                                                                     │
│ Channel(5) 目标渠道 (inherit_mode: override):                       │
│ {                                                                   │
│   models: ["gpt-4o"],                                              │
│   priority: 1                                                      │
│ }                                                                   │
│                                                                     │
│ 最终结果:                                                           │
│ {                                                                   │
│   base_url: "https://api.openai.com",  // 继承                     │
│   api_key: "sk-xxx",                    // 继承                     │
│   models: ["gpt-4o"],                   // 覆盖                     │
│   config: {timeout: 120, retry: 3},     // 继承                     │
│   weight: 3,                            // 继承中间渠道             │
│   priority: 1                           // 子渠道定义               │
│ }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

Step 3: 缓存解析结果
┌─────────────────────────────────────────────────────────────────────┐
│ Key: cdapi:channel:resolved:{channel_id}                           │
│ Value: {解析后的完整配置}                                           │
│ TTL: 300 秒                                                         │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 负载均衡配置

```
┌─────────────────────────────────────────────────────────────────────┐
│                      负载均衡配置                                    │
└─────────────────────────────────────────────────────────────────────┘

权重配置:
┌─────────────────┬──────────────────────────────────────────────────┐
│ 字段            │ 说明                                             │
├─────────────────┼──────────────────────────────────────────────────┤
│ weight          │ 负载均衡权重 (1-100)                             │
│                 │ 权重越高，被选中的概率越大                       │
│ priority        │ 优先级 (1-100)                                   │
│                 │ 数字越小优先级越高，同优先级内按权重分配         │
└─────────────────┴──────────────────────────────────────────────────┘

配置示例:
┌─────────────────┬────────┬──────────┬──────────────────────────────┐
│ 渠道            │ weight │ priority │ 说明                         │
├─────────────────┼────────┼──────────┼──────────────────────────────┤
│ OpenAI 主       │ 5      │ 1        │ 高优先级，高权重             │
│ OpenAI 备用     │ 3      │ 2        │ 次优先级，中等权重           │
│ Azure 主        │ 4      │ 1        │ 高优先级，较高权重           │
│ Anthropic       │ 2      │ 3        │ 低优先级，低权重             │
└─────────────────┴────────┴──────────┴──────────────────────────────┘

选择逻辑:
1. 按优先级分组 (priority 1 → 2 → 3)
2. 优先使用 priority=1 的渠道组
3. 在组内按权重随机选择
4. 如果 priority=1 组全部不可用，降级到 priority=2
```

## 五、渠道分组与标签

### 5.1 分组设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道分组设计                                  │
└─────────────────────────────────────────────────────────────────────┘

分组表:
CREATE TABLE channel_groups (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL COMMENT '分组名称',
    slug VARCHAR(100) NOT NULL UNIQUE COMMENT '分组标识',
    description TEXT NULL COMMENT '分组描述',
    config JSON NULL COMMENT '分组配置',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='渠道分组表';

渠道-分组关联表:
CREATE TABLE channel_group_pivot (
    channel_id BIGINT UNSIGNED NOT NULL,
    group_id BIGINT UNSIGNED NOT NULL,
    PRIMARY KEY (channel_id, group_id),
    FOREIGN KEY (channel_id) REFERENCES channels(id) ON DELETE CASCADE,
    FOREIGN KEY (group_id) REFERENCES channel_groups(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='渠道分组关联表';

分组用途:
├─── 按用途分组: production, development, testing
├─── 按地域分组: us-east, us-west, eu-west
├─── 按模型分组: gpt-4, claude, gemini
└─── 按租户分组: tenant-1, tenant-2
```

### 5.2 标签系统

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道标签系统                                  │
└─────────────────────────────────────────────────────────────────────┘

标签表:
CREATE TABLE channel_tags (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE COMMENT '标签名称',
    color VARCHAR(7) DEFAULT '#666666' COMMENT '标签颜色',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='渠道标签表';

渠道-标签关联表:
CREATE TABLE channel_tag_pivot (
    channel_id BIGINT UNSIGNED NOT NULL,
    tag_id BIGINT UNSIGNED NOT NULL,
    PRIMARY KEY (channel_id, tag_id),
    FOREIGN KEY (channel_id) REFERENCES channels(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES channel_tags(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='渠道标签关联表';

标签用途:
├─── 标记渠道特性: streaming, function-calling, vision
├─── 标记服务等级: premium, standard, economy
├─── 标记状态: beta, stable, deprecated
└─── 自定义标记

路由时使用标签:
{
    "channel_filters": {
        "tags": ["streaming", "premium"],
        "tags_mode": "all"  // all: 必须包含所有, any: 包含任一
    }
}
```

## 六、渠道 API

### 6.1 管理接口

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道管理 API                                  │
└─────────────────────────────────────────────────────────────────────┘

创建渠道:
POST /api/v1/channels
{
    "name": "OpenAI GPT-4 渠道",
    "slug": "openai-gpt4",
    "parent_id": null,
    "provider": "openai",
    "base_url": "https://api.openai.com",
    "api_key": "sk-xxx",
    "models": ["gpt-4", "gpt-4-turbo"],
    "weight": 5,
    "priority": 1,
    "config": {
        "timeout": {"request": 120},
        "retry": {"max_attempts": 3}
    },
    "inherit_mode": "merge",
    "tags": ["streaming", "premium"]
}

响应:
{
    "id": 1,
    "name": "OpenAI GPT-4 渠道",
    ...
    "resolved_config": {...}  // 解析后的完整配置
}

更新渠道:
PUT /api/v1/channels/{id}
{
    "weight": 3,
    "priority": 2,
    "status": "active"
}

获取渠道:
GET /api/v1/channels/{id}
响应包含: 基本信息 + 解析后的配置 + 继承链

获取渠道继承链:
GET /api/v1/channels/{id}/inheritance
响应:
{
    "chain": [
        {"id": 1, "name": "根渠道", "level": 1},
        {"id": 3, "name": "中间渠道", "level": 2},
        {"id": 5, "name": "目标渠道", "level": 3}
    ],
    "resolved_config": {...}
}

批量操作:
POST /api/v1/channels/batch
{
    "action": "enable",  // enable, disable, delete
    "channel_ids": [1, 2, 3]
}
```

## 七、渠道缓存策略

### 7.1 缓存设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道缓存设计                                  │
└─────────────────────────────────────────────────────────────────────┘

缓存层级:
├─── L1: 应用内存缓存 (进程内)
│    └─── 配置缓存、模型列表缓存
│
├─── L2: Redis 缓存 (分布式)
│    ├─── 渠道配置缓存
│    └─── 解析后的配置缓存
│
└─── L3: 数据库 (持久化)

缓存 Key 设计:
┌─────────────────────────────────────────────────────────────────────┐
│ Key                              │ Value              │ TTL        │
├──────────────────────────────────┼────────────────────┼────────────┤
│ cdapi:channel:{id}               │ 渠道基本信息       │ 300s       │
│ cdapi:channel:{id}:resolved      │ 解析后的配置       │ 300s       │
│ cdapi:channel:{id}:status        │ 状态信息           │ 60s        │
│ cdapi:channels:model:{model}     │ 支持该模型的渠道   │ 60s        │
│ cdapi:channels:active            │ 活跃渠道列表       │ 60s        │
└─────────────────────────────────────────────────────────────────────┘
```

### 7.2 缓存失效

```
┌─────────────────────────────────────────────────────────────────────┐
│                        缓存失效策略                                  │
└─────────────────────────────────────────────────────────────────────┘

主动失效:
├─── 渠道配置更新时
│    ├─── 删除 cdapi:channel:{id}
│    ├─── 删除 cdapi:channel:{id}:resolved
│    └─── 删除所有子渠道的 resolved 缓存
│
├─── 渠道状态变更时
│    └─── 删除 cdapi:channel:{id}:status
│
└─── 渠道删除时
     └─── 删除所有相关缓存

被动失效:
├─── TTL 过期自动删除
└─── 内存缓存 LRU 淘汰

缓存更新策略:
┌─────────────────┬──────────────────────────────────────────────────┐
│ 场景            │ 策略                                             │
├─────────────────┼──────────────────────────────────────────────────┤
│ 读取            │ 先查缓存，未命中则查库并写入缓存                 │
│ 更新            │ 先更新库，再删除缓存                             │
│ 批量更新        │ 先更新库，再批量删除缓存                         │
│ 继承链更新      │ 更新库后，删除所有受影响渠道的缓存               │
└─────────────────┴──────────────────────────────────────────────────┘
```

### 8.2 告警规则

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道告警规则                                  │
└─────────────────────────────────────────────────────────────────────┘

高错误率:
ALERT ChannelHighErrorRate
  IF sum(rate(cdapi_channel_errors_total[5m])) 
    / sum(rate(cdapi_channel_requests_total[5m])) > 0.1
  FOR 5m
  LABELS { severity: "warning" }

    summary: "渠道 {{ $labels.channel }} 熔断器已打开"
  }

高延迟:
ALERT ChannelHighLatency
  IF histogram_quantile(0.95, 
      rate(cdapi_channel_request_duration_seconds_bucket[5m])
  ) > 5
  FOR 5m
  LABELS { severity: "warning" }
```

## 九、最佳实践

### 9.1 渠道规划

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渠道规划建议                                  │
└─────────────────────────────────────────────────────────────────────┘

多渠道冗余:
├─── 每个模型至少配置 2 个渠道
├─── 主渠道 + 备用渠道
├─── 不同 Provider 互为备份
└─── 不同地域互为备份

继承层次:
├─── 根渠道: 定义基础配置 (API Key, base_url)
├─── 中间渠道: 定义模型分组
├─── 叶子渠道: 定义具体参数 (权重, 优先级)
└─── 建议最多 3 层继承

权重分配:
├─── 主渠道: weight=5, priority=1
├─── 备用渠道: weight=3, priority=2
├─── 降级渠道: weight=1, priority=3
└─── 根据实际性能调整
```
