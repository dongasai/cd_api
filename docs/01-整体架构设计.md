# 整体架构设计

## 一、系统概述

AI 代理系统是一个统一的 AI 服务网关，提供以下核心能力：

- **统一接口**：一个 API 端点接入所有 AI 服务
- **协议转换**：OpenAI 与 Anthropic 格式互转
- **智能路由**：多渠道负载均衡、故障自动切换
- **安全管控**：令牌权限管理、模型访问控制
- **数据洞察**：实时数据看板、用量统计、成本分析
- **多租户**：支持多租户隔离与配额管理

## 四、请求处理流程

```
1. 请求到达 Nginx
   │
   ▼
2. Laravel 接收请求
   │
   ▼
3. 中间件管道处理
   │
   ├─── CORS 中间件: 处理跨域请求
   │
   ├─── ResolveTenant 中间件:
   │    ├── 从 Header (X-Tenant-Slug) 或 Token 前缀解析租户
   │    ├── 验证租户状态
   │    └── 设置当前租户上下文
   │
   ├─── AuthenticateToken 中间件:
   │    ├── 提取 Authorization Header
   │    ├── 解析令牌前缀
   │    ├── 查询令牌记录
   │    ├── 验证令牌哈希
   │    ├── 检查令牌状态和过期时间
   │    └── 设置当前令牌上下文
   │
   ├─── RateLimiter 中间件:
   │    ├── 构建限流 Key
   │    ├── 检查 Redis 计数器
   │    └── 设置 RateLimit Headers
   │
   └─── AuditLogger 中间件:
        ├── 生成 request_id
        ├── 记录请求开始时间
        └── 注册响应后回调
   │
   ▼
4. 路由匹配 → 控制器处理
   │
   ▼
5. 调用 RouterService 选择渠道
   │
   ▼
6. 协议转换 (如需要)
   │
   ▼
7. 调用 Provider 发送请求
   │
   ▼
8. 返回响应给客户端
```
