## 二、Router Service (路由服务)

### 2.1 职责

- 根据请求模型选择合适的渠道
- 实现负载均衡策略
- 处理故障切换

### 2.2 类设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                         RouterService                                │
├─────────────────────────────────────────────────────────────────────┤
│ + route(model: string, context: RequestContext): Channel            │
│ + getAvailableChannels(model: string): Collection<Channel>          │

│ + markChannelSuccess(channel: Channel): void                        │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 路由流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                          路由选择流程                                │
└─────────────────────────────────────────────────────────────────────┘

输入: model="gpt-4", context={tenant_id: 1, token_id: 123}

1. 解析模型
   │
   ├─── 查询 model_mappings 表
   │    └─── alias="gpt-4" → actual_model="gpt-4-turbo-2024-04-09"
   │
   └─── 如果没有映射，使用原始模型名

2. 查找可用渠道
   │
   ├─── 查询 channels 表
   │    WHERE status = 'active'
   │    AND JSON_CONTAINS(models, '"gpt-4-turbo-2024-04-09"')
   │
   ├─── 应用租户/令牌的渠道限制
   │    └─── 过滤 allowed_channels
   │
   └─── 返回可用渠道列表

3. 负载均衡选择
   │
   ├─── 按优先级分组
   │    ├─── Priority 1: [Channel A, Channel B]
   │    └─── Priority 2: [Channel C]
   │
   ├─── 选择最高优先级组
   │    └─── Priority 1 组
   │
   └─── 在组内使用加权随机
        ├─── Channel A: weight=5, 范围 [1-5]
        ├─── Channel B: weight=3, 范围 [6-8]
        ├─── 随机数 = 7
        └─── 选择 Channel B

4. 返回选中的渠道
   └─── Channel B
```

### 2.4 负载均衡算法

#### 加权随机算法

```
输入:
  Channel A: weight=5
  Channel B: weight=3
  Channel C: weight=2

算法:
1. 计算总权重: total = 5 + 3 + 2 = 10
2. 生成随机数: random = rand(1, total) = 7
3. 遍历累加:
   - Channel A: cumulative = 5, random(7) > 5, 继续
   - Channel B: cumulative = 5 + 3 = 8, random(7) <= 8, 选择 Channel B

返回: Channel B
```

#### 最小连接数算法

```
输入:
  Channel A: active_connections=10, weight=5
  Channel B: active_connections=5, weight=3
  Channel C: active_connections=8, weight=2

算法:
1. 计算加权连接数:
   - Channel A: 10 / 5 = 2.0
   - Channel B: 5 / 3 = 1.67
   - Channel C: 8 / 2 = 4.0

2. 选择最小值: Channel B (1.67)

返回: Channel B
```

### 2.5 故障切换策略

```
┌─────────────────────────────────────────────────────────────────────┐
│                          故障判断规则                                │
└─────────────────────────────────────────────────────────────────────┘

不重试 (客户端错误):
├─── 400 Bad Request
├─── 401 Unauthorized (API Key 无效)
├─── 403 Forbidden
├─── 422 Unprocessable Entity
└─── 直接返回错误给客户端

重试 (服务端/网络错误):
├─── 429 Too Many Requests
├─── 500 Internal Server Error
├─── 502 Bad Gateway
├─── 503 Service Unavailable
├─── 504 Gateway Timeout
├─── Connection Timeout
└─── Network Error

┌─────────────────────────────────────────────────────────────────────┐
│                          故障切换流程                                │
└─────────────────────────────────────────────────────────────────────┘

请求失败 (可重试错误)
    │
    ▼
记录失败
    │
    ├─── channel.failure_count++
    └─── channel.last_failure_at = now()

    │
    ▼
检查重试次数
    │
    ├─── retry_count < max_retries (默认 3)
    │    │
    │    ▼
    │    选择备用渠道
    │    │
    │    ├─── 排除已失败的渠道
    │    └─── 从剩余渠道中选择 (负载均衡)
    │    │
    │    ▼
    │    重试请求
    │    │
    │    └─── retry_count++
    │
    └─── retry_count >= max_retries
         └─── 返回最后一个错误给客户端
```

## 四、SSE 流式响应处理

### 4.1 SSE 基础

```
┌─────────────────────────────────────────────────────────────────────┐
│                     Server-Sent Events 格式                          │
└─────────────────────────────────────────────────────────────────────┘

HTTP Headers:
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no  (Nginx 禁用缓冲)

数据格式:
data: {json_data}\n\n

或带事件类型:
event: message_start\ndata: {...}\n\n
```

### 4.2 流式响应处理

```
┌─────────────────────────────────────────────────────────────────────┐
│                          SSEStreamer                                 │
├─────────────────────────────────────────────────────────────────────┤
│ + stream(callable $generator): Response                             │
│ + sendEvent(string $event, string $data): void                      │
│ + sendData(string $data): void                                      │
│ + sendDone(): void                                                  │
└─────────────────────────────────────────────────────────────────────┘

处理流程:
1. 接收客户端流式请求
2. 建立到上游的流式连接
3. 逐块读取上游响应
4. 转换格式 (如需要)
5. 实时发送给客户端
6. 处理客户端断开连接
```

### 4.3 Laravel 实现方式

```
方式一: StreamedResponse
return response()->stream(function () use ($provider, $request) {
    $generator = $provider->sendStream($request);
    foreach ($generator as $chunk) {
        echo "data: " . json_encode($chunk) . "\n\n";
        ob_flush();
        flush();
    }
    echo "data: [DONE]\n\n";
}, 200, [
    'Content-Type' => 'text/event-stream',
    'Cache-Control' => 'no-cache',
    'X-Accel-Buffering' => 'no',
]);

方式二: 自定义 Response 类
class SSEResponse extends Response
{
    public function __construct(callable $callback)
    {
        parent::__construct();
        $this->headers->set('Content-Type', 'text/event-stream');
        $this->headers->set('Cache-Control', 'no-cache');
        $this->headers->set('X-Accel-Buffering', 'no');
        $this->callback = $callback;
    }
}
```
