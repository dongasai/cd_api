```php
// ❌ 错误示例 - ChannelController.php

    'active' => '正常',
```

#### 菜单翻译问题
- 菜单标题使用数据库存储，需要特殊处理
- 英文语言文件中存在重复的菜单翻译方式

## 二、多语言方案架构

### 2.1 架构设计原则

1. **统一翻译键**：所有用户可见文本必须通过 `trans()` 函数获取
2. **分层管理**：通用翻译与业务翻译分离
3. **开发友好**：提供便捷的翻译辅助函数
4. **可维护性**：翻译文件结构清晰，易于扩展

### 2.2 语言文件结构设计

```
laravel/lang/
├── zh_CN/
│   ├── admin.php          # 后台通用翻译（框架文本）
│   ├── models.php         # 模型相关翻译（新增）
│   ├── fields.php         # 字段名称翻译（新增）
│   ├── statuses.php       # 状态值翻译（新增）
│   └── validation.php     # 验证消息翻译
└── en/
    ├── admin.php
    ├── models.php
    ├── fields.php
    ├── statuses.php
    └── validation.php
```

#### 语言文件职责划分

| 文件 | 职责 | 示例 |
|------|------|------|
| `admin.php` | 框架通用文本、操作按钮 | 保存、删除、编辑、列表 |
| `models.php` | 模型名称、控制器标题 | 渠道管理、Coding账户 |
| `fields.php` | 数据库字段名称 | 渠道名称、账户名称、状态 |
| `statuses.php` | 状态值、枚举值 | 正常、禁用、维护中 |

### 2.3 翻译键命名规范

#### 通用命名规则
```php
// 格式：{文件名}.{模块}.{具体文本}

// 示例：
trans('models.channel.title')           // 渠道管理
trans('fields.channel.name')            // 渠道名称
trans('statuses.channel.active')        // 正常
trans('admin.save')                     // 保存
```

#### 字段翻译规范
```php
// 按模型分组
trans('fields.channel.name')            // 渠道名称
trans('fields.channel.slug')            // 标识符
trans('fields.coding_account.name')     // 账户名称
trans('fields.coding_account.status')   // 状态

// 通用字段
trans('fields.common.id')               // ID
trans('fields.common.created_at')       // 创建时间
trans('fields.common.updated_at')       // 更新时间
```

#### 状态值翻译规范
```php
// 按模块分组
trans('statuses.channel.active')        // 正常
trans('statuses.channel.disabled')      // 禁用
trans('statuses.channel.maintenance')   // 维护中

trans('statuses.coding_account.active')    // 活跃
trans('statuses.coding_account.exhausted') // 已耗尽
```

```php
<?php

namespace App\Models;

use Dcat\Admin\Models\Administrator as BaseAdministrator;

class Administrator extends BaseAdministrator
{
    /**
     * 获取翻译的字段标签
     *

     * @return string

    public static function getFieldLabel(string $field): string
    {
        return trans("fields.administrator.{$field}");
    }

    /**
     * 获取翻译的状态标签
     *
     * @param  string  $status
     * @return string
     */
    public static function getStatusLabel(string $status): string
    {
        return trans("statuses.administrator.{$status}");
    }
}
```

### 3.4 第四阶段：菜单多语言

#### 3.4.1 菜单翻译方案

**方案一：使用翻译键（推荐）**

修改菜单数据库记录，使用翻译键代替硬编码文本：

```sql
-- 更新菜单标题为翻译键
UPDATE admin_menu SET title = 'menu_titles.dashboard' WHERE id = 1;
UPDATE admin_menu SET title = 'menu_titles.api_keys' WHERE id = 2;
UPDATE admin_menu SET title = 'menu_titles.channels' WHERE id = 3;
```

**菜单渲染时自动翻译**：

在 `config/admin.php` 中配置菜单翻译回调：

```php
'menu' => [
    'translator' => function ($title) {
        // 如果是翻译键格式，则翻译
        if (str_starts_with($title, 'menu_titles.')) {
            return trans('admin.'.$title);
        }
        // 否则尝试直接翻译
        return trans('admin.'.$title) ?? $title;
    },
],
```

**方案二：动态菜单翻译**

创建菜单中间件，自动翻译菜单项：

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Dcat\Admin\Admin;
use Illuminate\Http\Request;

class TranslateMenuMiddleware
{
    public function handle(Request $request, Closure $next)
    {
        // 获取菜单数据
        $menu = Admin::menu();

        // 递归翻译菜单项
        $this->translateMenu($menu);

        return $next($request);
    }

    private function translateMenu(&$menu)
    {
        foreach ($menu as &$item) {
            // 翻译菜单标题
            if (isset($item['title'])) {
                $item['title'] = $this->translateTitle($item['title']);
            }

            // 递归处理子菜单
            if (isset($item['children'])) {
                $this->translateMenu($item['children']);
            }
        }
    }

    private function translateTitle($title)
    {
        // 尝试从 menu_titles 翻译
        $translated = trans('admin.menu_titles.'.$title);
        if ($translated !== 'admin.menu_titles.'.$title) {
            return $translated;
        }

        // 尝试直接翻译（兼容旧的中文名称）
        return trans('admin.'.$title) ?? $title;
    }
}
```

## 六、注意事项

### 6.1 开发注意事项

1. **不要遗漏任何用户可见文本**
   - 列表字段标签
   - 表单字段标签
   - 筛选器字段标签
   - 操作按钮文本
   - 提示信息文本
   - 错误消息文本

2. **保持翻译一致性**
   - 同一字段在不同页面使用相同翻译
   - 相同概念使用统一术语

3. **注意翻译上下文**
   ```php
   // 根据上下文选择合适的翻译
   trans('fields.channel.name')     // 作为字段标签
   trans('models.channel.singular') // 作为模型名称
   ```

4. **处理复数形式**
   ```php
   // 在语言文件中定义单复数
   'model' => [
       'singular' => 'Model',
       'plural' => 'Models',
   ],

   // 使用时选择合适的形式
   trans('models.channel.singular') // 单个渠道
   trans('models.channel.plural')   // 多个渠道
   ```

### 6.2 部署注意事项

1. **数据库迁移**
   - 确保 `admin_users.language` 字段已添加
   - 设置合理的默认值（`zh_CN`）

2. **缓存清理**
   - 部署后执行：`php artisan optimize:clear`
   - 清理语言缓存：`php artisan cache:clear`

3. **权限检查**
   - 确保语言文件可读
   - 检查文件权限：`chmod -R 755 laravel/lang`

### 6.3 维护注意事项

1. **新增字段**
   - 添加新字段时，同步更新语言文件
   - 在 `fields.php` 中添加字段翻译

2. **新增模型**
   - 创建新模型时，在 `models.php` 中添加翻译
   - 在控制器中使用 `trans('models.xxx.title')`

3. **新增状态**
   - 添加新状态时，在 `statuses.php` 中添加翻译
   - 更新控制器中的状态映射

## 七、FAQ

### 7.1 如何处理动态内容？

**问题**：部分内容是从数据库读取的，如何翻译？

**方案**：使用 JSON 字段存储多语言内容

```php
// 数据库表结构
Schema::create('articles', function (Blueprint $table) {
    $table->id();
    $table->json('title'); // {"zh_CN": "标题", "en": "Title"}
    $table->timestamps();
});

// 模型方法
class Article extends Model
{
    protected $casts = [
        'title' => 'array',
    ];

    public function getLocalizedTitle(): string
    {
        return $this->title[app()->getLocale()] ?? $this->title['zh_CN'];
    }
}
```

### 7.2 如何处理日期格式？

**问题**：不同语言的日期格式不同

**方案**：使用 Carbon 的本地化功能

```php
// Carbon 会自动根据应用语言设置调整格式
$date = Carbon::now();
echo $date->isoFormat('LLLL');

// 中文：2026年3月18日星期三 上午12:54
// 英文：Wednesday, March 18, 2026 12:54 AM
```

### 7.3 如何处理图片中的文字？

**问题**：图片中包含文字，无法翻译

**方案**：
1. 避免在图片中嵌入文字
2. 使用 SVG 图标代替文字图片
3. 为不同语言准备不同图片

### 7.4 如何处理第三方扩展？

**问题**：第三方扩展的文本如何翻译？

**方案**：
1. 检查扩展是否支持多语言
2. 如果不支持，可以覆盖扩展的视图文件
3. 在项目语言文件中添加扩展的翻译
