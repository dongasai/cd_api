# 数据库设计

## 设计原则

本数据库设计遵循以下原则，以确保高性能和可维护性：

1. **无外键约束** - 所有关联关系在应用层处理，避免数据库层面的性能损耗
2. **无 CHECK 约束** - 数据验证在应用层完成，提高灵活性
3. **无触发器** - 业务逻辑在应用层实现，便于调试和维护
4. **无视图** - 直接查询表，避免额外的抽象层
5. **字段命名规范** - 使用小写字母和下划线分隔，避免使用数据库保留字

**关联关系处理方式：**
- 应用层通过代码维护数据一致性
- 使用事务确保关联操作的原子性
- 定期运行数据一致性检查任务

### 2.2 api_keys (API密钥表)

```sql
CREATE TABLE api_keys (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL COMMENT 'API密钥名称',
    key_hash VARCHAR(255) NOT NULL COMMENT 'API密钥哈希',
    key_prefix VARCHAR(20) NOT NULL COMMENT 'API密钥前缀',
    permissions JSON DEFAULT NULL COMMENT '权限配置',
    allowed_models JSON DEFAULT NULL COMMENT '允许的模型列表',
    rate_limit JSON DEFAULT NULL COMMENT '速率限制配置',
    expires_at TIMESTAMP NULL COMMENT '过期时间',
    last_used_at TIMESTAMP NULL COMMENT '最后使用时间',
    status ENUM('active', 'revoked', 'expired') DEFAULT 'active' COMMENT '状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_key_prefix (key_prefix)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='API密钥表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| key_hash | VARCHAR(255) | bcrypt 哈希后的API密钥 |
| key_prefix | VARCHAR(20) | API密钥前缀，如 "sk"、"sk_live" |
| permissions | JSON | 权限配置 |
| allowed_models | JSON | 允许访问的模型，NULL 表示无限制 |
| rate_limit | JSON | API密钥级别速率限制 |

**permissions JSON 结构：**
```json
{
    "endpoints": ["chat", "completions", "embeddings"],
    "max_tokens_per_request": 4000,
    "allowed_channels": [1, 2, 3]
}
```

**rate_limit JSON 结构：**
```json
{
    "requests_per_minute": 60,
    "tokens_per_minute": 10000,
    "requests_per_day": 10000
}
```

### 2.3 channels (渠道表)

```sql
CREATE TABLE channels (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    
    -- 继承关系
    parent_id BIGINT UNSIGNED NULL COMMENT '父渠道 ID',
    inherit_mode ENUM('merge', 'override', 'extend') DEFAULT 'merge' COMMENT '继承模式',
    
    -- 基础信息
    name VARCHAR(255) NOT NULL COMMENT '渠道名称',
    slug VARCHAR(100) NULL COMMENT '渠道标识',
    provider VARCHAR(50) NOT NULL COMMENT '提供商类型',
    
    -- 连接配置 (可继承)
    base_url VARCHAR(500) NULL COMMENT 'API 基础 URL',
    api_key TEXT NULL COMMENT '加密的 API Key',
    api_key_hash VARCHAR(64) NULL COMMENT 'API Key 指纹 (SHA256前8位)',
    
    -- 模型配置 (可继承)
    models JSON NULL COMMENT '支持的模型列表',
    
    -- 负载均衡
    weight INT UNSIGNED DEFAULT 1 COMMENT '负载均衡权重 (1-100)',
    priority INT UNSIGNED DEFAULT 1 COMMENT '优先级 (越小越优先)',
    
    -- 状态管理
    status ENUM('active', 'disabled', 'maintenance') DEFAULT 'active' COMMENT '运营状态',
    health_status ENUM('healthy', 'unhealthy', 'unknown') DEFAULT 'unknown' COMMENT '健康状态',
    failure_count INT UNSIGNED DEFAULT 0 COMMENT '连续失败次数',
    success_count INT UNSIGNED DEFAULT 0 COMMENT '连续成功次数',
    
    -- 时间记录
    last_check_at TIMESTAMP NULL COMMENT '最后健康检查时间',
    last_failure_at TIMESTAMP NULL COMMENT '最后失败时间',
    last_success_at TIMESTAMP NULL COMMENT '最后成功时间',
    
    -- 统计信息 (冗余存储)
    total_requests BIGINT UNSIGNED DEFAULT 0 COMMENT '总请求数',
    total_tokens BIGINT UNSIGNED DEFAULT 0 COMMENT '总 Token 数',
    total_cost DECIMAL(12, 6) DEFAULT 0 COMMENT '总成本',
    avg_latency_ms INT UNSIGNED DEFAULT 0 COMMENT '平均延迟',
    success_rate DECIMAL(5, 4) DEFAULT 1.0000 COMMENT '成功率',
    
    -- 高级配置 (可继承)
    config JSON NULL COMMENT '额外配置',
    
    -- 元数据
    description TEXT NULL COMMENT '渠道描述',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 索引
    INDEX idx_parent (parent_id),
    INDEX idx_slug (slug),
    INDEX idx_provider_status (provider, status),
    INDEX idx_status_health (status, health_status),
    INDEX idx_status_priority (status, priority)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='渠道表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| parent_id | BIGINT | 父渠道 ID，支持继承机制 |
| inherit_mode | ENUM | 继承模式：merge(合并)、override(覆盖)、extend(扩展) |
| slug | VARCHAR(100) | 渠道唯一标识，用于 API 引用 |
| provider | VARCHAR(50) | openai, anthropic, azure, google, custom |
| api_key_hash | VARCHAR(64) | API Key 指纹，用于日志脱敏显示 |
| weight | INT | 负载均衡权重 (1-100)，权重越高被选中概率越大 |
| priority | INT | 优先级 (1-100)，数字越小优先级越高 |
| health_status | ENUM | 健康状态，用于故障切换判断 |
| success_rate | DECIMAL(5,4) | 成功率，用于智能路由决策 |

**provider 枚举值：**

| 值 | 说明 | 健康检查方式 |
|----|------|-------------|
| openai | OpenAI 官方 API | GET /v1/models |
| azure | Azure OpenAI Service | GET /deployments |
| anthropic | Anthropic Claude API | POST /v1/messages |
| google | Google Gemini API | GET /v1/models |
| custom | 自定义 OpenAI 兼容 API | 配置的 health_check.endpoint |

**models JSON 结构：**
```json
["gpt-4", "gpt-4-turbo", "gpt-3.5-turbo"]
```

**config JSON 结构：**
```json
{
    "timeout": {
        "connect": 5,
        "request": 120,
        "read": 30
    },
    "retry": {
        "max_attempts": 3,
        "delay_ms": 1000,
        "multiplier": 2,
        "max_delay_ms": 10000
    },
    "rate_limit": {
        "requests_per_minute": 500,
        "tokens_per_minute": 100000
    },
    "circuit_breaker": {
        "enabled": true,
        "failure_threshold": 5,
        "recovery_timeout": 60,
        "half_open_requests": 3
    },
    "health_check": {
        "enabled": true,
        "interval": 60,
        "endpoint": "/v1/models",
        "timeout": 10,
        "expected_status": 200
    },
    "proxy": {
        "enabled": false,
        "url": "http://proxy:8080"
    },
    "headers": {
        "X-Custom-Header": "value"
    }

```

**继承模式说明：**

| 模式 | 数组类型 | 对象类型 | 标量类型 | 适用场景 |
|------|----------|----------|----------|----------|
| merge | 合并去重 | 深度合并 | 子覆盖父 | 扩展父渠道配置 |
| override | 使用子值 | 使用子值 | 使用子值 | 完全自定义配置 |
| extend | 追加到父 | 合并到父 | 子覆盖父 | 保留父配置并添加 |

```sql
CREATE TABLE audit_logs (

    cost DECIMAL(10, 6) DEFAULT 0.000000 COMMENT '成本 (美元)',
    quota DECIMAL(10, 6) DEFAULT 0.000000 COMMENT '配额消耗点数',
    billing_source VARCHAR(50) DEFAULT 'wallet' COMMENT '计费来源: wallet, quota, trial',

    -- 响应信息
    status_code SMALLINT UNSIGNED NULL COMMENT 'HTTP 状态码',
    latency_ms INT UNSIGNED DEFAULT 0 COMMENT '总响应延迟 (毫秒)',
    first_token_ms INT UNSIGNED DEFAULT 0 COMMENT '首 token 延迟 (毫秒)',
    is_stream TINYINT(1) DEFAULT 0 COMMENT '是否流式请求',
```

```sql
    PARTITION p202403 VALUES LESS THAN (UNIX_TIMESTAMP('2024-04-01')),
    PARTITION pmax VALUES LESS THAN MAXVALUE
);

-- PostgreSQL 分区示例
CREATE TABLE audit_logs (
    ...

CREATE TABLE audit_logs_2024_01 PARTITION OF audit_logs

CREATE TABLE audit_logs_2024_02 PARTITION OF audit_logs
    FOR VALUES FROM ('2024-02-01') TO ('2024-03-01');
```

### 2.9 request_logs (请求日志表)


**字段说明：**

## 四、数据归档策略

```
┌─────────────────────────────────────────────────────────────────────┐
│                          数据生命周期                                │
└─────────────────────────────────────────────────────────────────────┘

审计日志 (audit_logs):
├─── 热数据 (0-7 天): 主数据库
├─── 温数据 (7-90 天): 归档表 / 时序数据库
└─── 冷数据 (>90 天): 对象存储 / 数据仓库

请求日志 (request_logs):
├─── 热数据 (0-3 天): 主数据库 (高频查询)
├─── 温数据 (3-30 天): 归档表
└─── 冷数据 (>30 天): 对象存储 (压缩存储)

响应日志 (response_logs):
├─── 热数据 (0-3 天): 主数据库
├─── 温数据 (3-30 天): 归档表
└─── 冷数据 (>30 天): 对象存储

渠道转发日志 (channel_forwarding_logs):
├─── 热数据 (0-7 天): 主数据库
├─── 温数据 (7-30 天): 归档表
└─── 冷数据 (>30 天): 对象存储

归档流程:
1. 定时任务每日执行
2. 将过期数据导出为 Parquet 格式
3. 上传到对象存储 (S3/OSS)
4. 删除已归档数据
5. 更新归档索引表

存储优化建议:
├─── request_logs.body_text / response_logs.body_text: 
│    └── 使用压缩算法 (zstd/gzip) 减少存储空间
├─── request_logs.messages / response_logs.generated_text:
│    └── 可选择性存储，根据 log_storage_config 配置
└─── channel_forwarding_logs.upstream_body_snippet:
     └── 仅存储前 1000 字符，避免大量数据存储
```

## 五、Redis 缓存设计

### 5.1 缓存 Key 设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                          缓存 Key 规范                               │
└─────────────────────────────────────────────────────────────────────┘

API密钥验证缓存:
Key:   cdapi:api_key:validate:{key_prefix}:{hash_prefix}
Value: {api_key_id, permissions, ...}
TTL:   60s

渠道列表缓存:
Key:   cdapi:channels:model:{model}
Value: [{channel_id, weight, priority, ...}, ...]
TTL:   60s

统计缓存:
Key:   cdapi:stats:{date}
Value: {requests, tokens, cost, ...}
TTL:   实时更新，每日过期

速率限制计数:
Key:   cdapi:rate:{identifier}
Value: ZSET (滑动窗口)
TTL:   根据窗口大小
```

### 5.2 缓存失效策略

```
API密钥更新:
├─── 更新数据库
└─── 删除 cdapi:api_key:validate:{key_prefix}:*

渠道更新:
├─── 更新数据库
└─── 删除 cdapi:channels:model:*
```
