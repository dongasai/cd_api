## 二、表结构详细设计

### 2.1 tenants (租户表)

```sql
CREATE TABLE tenants (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL COMMENT '租户名称',
    slug VARCHAR(100) NOT NULL UNIQUE COMMENT '租户标识',
    settings JSON DEFAULT NULL COMMENT '配置 (模型限制、速率限制等)',
    status ENUM('active', 'suspended', 'deleted') DEFAULT 'active' COMMENT '状态',
    quota_limit DECIMAL(10, 2) DEFAULT NULL COMMENT '配额限制 (美元)',
    quota_used DECIMAL(10, 2) DEFAULT 0.00 COMMENT '已用配额 (美元)',
    quota_reset_at TIMESTAMP NULL COMMENT '配额重置时间',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_slug (slug),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='租户表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(255) | 租户名称，如 "Acme Corp" |
| slug | VARCHAR(100) | 租户标识，用于 API 识别，如 "acme" |
| settings | JSON | 租户级别配置 |
| status | ENUM | active: 正常, suspended: 暂停, deleted: 已删除 |
| quota_limit | DECIMAL | 配额限制，NULL 表示无限制 |
| quota_used | DECIMAL | 已使用配额 |
| quota_reset_at | TIMESTAMP | 配额重置时间（月度重置） |

**settings JSON 结构：**
```json
{
    "allowed_models": ["gpt-4", "claude-3-opus"],
    "default_rate_limit": {
        "requests_per_minute": 60,
        "tokens_per_minute": 10000
    },
    "features": {
        "streaming": true,
        "embeddings": true
    }
}
```

### 2.2 tokens (令牌表)

```sql
CREATE TABLE tokens (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    tenant_id BIGINT UNSIGNED NOT NULL COMMENT '租户 ID',
    name VARCHAR(255) NOT NULL COMMENT '令牌名称',
    key_hash VARCHAR(255) NOT NULL COMMENT '令牌哈希',
    key_prefix VARCHAR(20) NOT NULL COMMENT '令牌前缀',
    permissions JSON DEFAULT NULL COMMENT '权限配置',
    allowed_models JSON DEFAULT NULL COMMENT '允许的模型列表',
    rate_limit JSON DEFAULT NULL COMMENT '速率限制配置',
    expires_at TIMESTAMP NULL COMMENT '过期时间',
    last_used_at TIMESTAMP NULL COMMENT '最后使用时间',
    status ENUM('active', 'revoked', 'expired') DEFAULT 'active' COMMENT '状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_tenant_status (tenant_id, status),
    INDEX idx_key_prefix (key_prefix),
    FOREIGN KEY (tenant_id) REFERENCES tenants(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='令牌表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| key_hash | VARCHAR(255) | bcrypt 哈希后的令牌 |
| key_prefix | VARCHAR(20) | 令牌前缀，如 "sk"、"sk_live" |
| permissions | JSON | 权限配置 |
| allowed_models | JSON | 允许访问的模型，NULL 表示继承租户配置 |
| rate_limit | JSON | 令牌级别速率限制 |

**permissions JSON 结构：**
```json
{
    "endpoints": ["chat", "completions", "embeddings"],
    "max_tokens_per_request": 4000,
    "allowed_channels": [1, 2, 3]
}
```

**rate_limit JSON 结构：**
```json
{
    "requests_per_minute": 60,
    "tokens_per_minute": 10000,
    "requests_per_day": 10000
}
```

### 2.3 channels (渠道表)

```sql
CREATE TABLE channels (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL COMMENT '渠道名称',
    provider VARCHAR(50) NOT NULL COMMENT '提供商类型',
    base_url VARCHAR(500) NOT NULL COMMENT 'API 基础 URL',
    api_key TEXT NOT NULL COMMENT '加密的 API Key',
    models JSON NOT NULL COMMENT '支持的模型列表',
    weight INT UNSIGNED DEFAULT 1 COMMENT '负载均衡权重',
    priority INT UNSIGNED DEFAULT 1 COMMENT '优先级 (越小越优先)',
    status ENUM('active', 'disabled', 'maintenance') DEFAULT 'active' COMMENT '状态',
    health_status ENUM('healthy', 'unhealthy', 'unknown') DEFAULT 'unknown' COMMENT '健康状态',
    failure_count INT UNSIGNED DEFAULT 0 COMMENT '连续失败次数',
    last_check_at TIMESTAMP NULL COMMENT '最后健康检查时间',
    last_failure_at TIMESTAMP NULL COMMENT '最后失败时间',
    config JSON DEFAULT NULL COMMENT '额外配置',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_provider_status (provider, status),
    INDEX idx_status_health (status, health_status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='渠道表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| provider | VARCHAR(50) | openai, anthropic, azure, custom |
| weight | INT | 负载均衡权重，默认 1 |
| priority | INT | 优先级，数字越小优先级越高 |
| health_status | ENUM | 健康状态，用于故障切换判断 |
| failure_count | INT | 连续失败次数，超过阈值标记为 unhealthy |
| config | JSON | 额外配置，如超时时间、重试策略等 |

**models JSON 结构：**
```json
["gpt-4", "gpt-4-turbo", "gpt-3.5-turbo"]
```

**config JSON 结构：**
```json
{
    "timeout": 120,
    "connect_timeout": 5,
    "max_retries": 3,
    "retry_delay": 1000,
    "failure_threshold": 3,
    "recovery_timeout": 300
}
```

### 2.4 audit_logs (审计日志表)

```sql
CREATE TABLE audit_logs (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    tenant_id BIGINT UNSIGNED NOT NULL COMMENT '租户 ID',
    token_id BIGINT UNSIGNED NULL COMMENT '令牌 ID',
    channel_id BIGINT UNSIGNED NULL COMMENT '渠道 ID',
    request_id VARCHAR(36) NOT NULL COMMENT '请求唯一标识',
    method VARCHAR(10) NOT NULL COMMENT 'HTTP 方法',
    path VARCHAR(500) NOT NULL COMMENT '请求路径',
    model VARCHAR(100) NULL COMMENT '使用的模型',
    input_tokens INT UNSIGNED DEFAULT 0 COMMENT '输入 token 数',
    output_tokens INT UNSIGNED DEFAULT 0 COMMENT '输出 token 数',
    total_tokens INT UNSIGNED DEFAULT 0 COMMENT '总 token 数',
    cost DECIMAL(10, 6) DEFAULT 0.000000 COMMENT '成本 (美元)',
    latency_ms INT UNSIGNED DEFAULT 0 COMMENT '响应延迟 (毫秒)',
    status_code SMALLINT UNSIGNED NULL COMMENT 'HTTP 状态码',
    error_message TEXT NULL COMMENT '错误信息',
    client_ip VARCHAR(45) NULL COMMENT '客户端 IP',
    user_agent VARCHAR(500) NULL COMMENT 'User Agent',
    metadata JSON DEFAULT NULL COMMENT '额外元数据',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_tenant_created (tenant_id, created_at),
    INDEX idx_token_created (token_id, created_at),
    INDEX idx_request_id (request_id),
    INDEX idx_channel_created (channel_id, created_at),
    FOREIGN KEY (tenant_id) REFERENCES tenants(id) ON DELETE CASCADE,
    FOREIGN KEY (token_id) REFERENCES tokens(id) ON DELETE SET NULL,
    FOREIGN KEY (channel_id) REFERENCES channels(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='审计日志表';
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| request_id | VARCHAR(36) | UUID 格式的请求唯一标识 |
| cost | DECIMAL(10, 6) | 本次请求成本，精确到微美元 |
| metadata | JSON | 额外元数据，如请求参数摘要 |

**metadata JSON 结构：**
```json
{
    "stream": true,
    "temperature": 0.7,
    "max_tokens": 100,
    "message_count": 3,
    "finish_reason": "stop"
}
```

### 2.5 model_mappings (模型映射表)

```sql
CREATE TABLE model_mappings (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    alias VARCHAR(100) NOT NULL UNIQUE COMMENT '模型别名 (对外展示)',
    actual_model VARCHAR(100) NOT NULL COMMENT '实际模型名称',
    channel_id BIGINT UNSIGNED NULL COMMENT '默认渠道 ID',
    enabled BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_alias (alias),
    INDEX idx_enabled (enabled),
    FOREIGN KEY (channel_id) REFERENCES channels(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='模型映射表';
```

**使用场景：**

| alias | actual_model | 说明 |
|-------|--------------|------|
| gpt-4 | gpt-4-turbo-2024-04-09 | 指向最新版本 |
| claude-3 | claude-3-opus-20240229 | 简化模型名 |
| my-custom-model | gpt-4 | 自定义模型名 |

## 三、索引设计

### 3.1 高频查询索引

```sql
-- 令牌查询
tokens: idx_key_prefix (key_prefix)  -- 快速定位令牌
tokens: idx_tenant_status (tenant_id, status)  -- 租户令牌列表

-- 审计日志查询
audit_logs: idx_tenant_created (tenant_id, created_at)  -- 租户日志查询
audit_logs: idx_token_created (token_id, created_at)  -- 令牌日志查询
audit_logs: idx_request_id (request_id)  -- 请求追踪

-- 渠道查询
channels: idx_provider_status (provider, status)  -- 按提供商筛选
channels: idx_status_health (status, health_status)  -- 健康渠道筛选
```

### 3.2 分区策略

审计日志表数据量大，建议按月分区：

```sql
-- PostgreSQL 分区示例
CREATE TABLE audit_logs (
    ...
) PARTITION BY RANGE (created_at);

CREATE TABLE audit_logs_2024_01 PARTITION OF audit_logs
    FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');

CREATE TABLE audit_logs_2024_02 PARTITION OF audit_logs
    FOR VALUES FROM ('2024-02-01') TO ('2024-03-01');
```

## 四、数据归档策略

```
┌─────────────────────────────────────────────────────────────────────┐
│                          数据生命周期                                │
└─────────────────────────────────────────────────────────────────────┘

审计日志:
├─── 热数据 (0-7 天): 主数据库
├─── 温数据 (7-90 天): 归档表 / 时序数据库
└─── 冷数据 (>90 天): 对象存储 / 数据仓库

归档流程:
1. 定时任务每日执行
2. 将 90 天前的数据导出为 Parquet 格式
3. 上传到对象存储 (S3/OSS)
4. 删除已归档数据
5. 更新归档索引表
```

## 五、Redis 缓存设计

### 5.1 缓存 Key 设计

```
┌─────────────────────────────────────────────────────────────────────┐
│                          缓存 Key 规范                               │
└─────────────────────────────────────────────────────────────────────┘

令牌验证缓存:
Key:   cdapi:token:validate:{key_prefix}:{hash_prefix}
Value: {tenant_id, token_id, permissions, ...}
TTL:   60s

租户配置缓存:
Key:   cdapi:tenant:config:{tenant_id}
Value: {settings, quota_limit, quota_used, ...}
TTL:   300s

渠道列表缓存:
Key:   cdapi:channels:model:{model}
Value: [{channel_id, weight, priority, ...}, ...]
TTL:   60s

统计缓存:
Key:   cdapi:stats:{tenant_id}:{date}
Value: {requests, tokens, cost, ...}
TTL:   实时更新，每日过期

速率限制计数:
Key:   cdapi:rate:{identifier}
Value: ZSET (滑动窗口)
TTL:   根据窗口大小
```

### 5.2 缓存失效策略

```
令牌更新:
├─── 更新数据库
└─── 删除 cdapi:token:validate:{key_prefix}:*

租户更新:
├─── 更新数据库
└─── 删除 cdapi:tenant:config:{tenant_id}

渠道更新:
├─── 更新数据库
└─── 删除 cdapi:channels:model:*
```
