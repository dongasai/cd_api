### 2. 修改菜单标题为翻译键

在数据库中将菜单的 `title` 字段改为翻译键格式 `admin.menu_titles.xxx`：

```php
// 示例：将 "仪表盘" 改为 "admin.menu_titles.dashboard"
UPDATE admin_menu SET title = 'admin.menu_titles.dashboard' WHERE id = 50;
UPDATE admin_menu SET title = 'admin.menu_titles.api_keys' WHERE id = 52;
// ... 其他菜单项
```

## 方法二：创建菜单迁移脚本自动更新

创建一个迁移文件来更新菜单标题：

```bash
php artisan make:migration update_menu_titles_for_multilanguage --path=database/migrations
```

## 方法三：动态菜单翻译（不修改数据库）

如果不想修改数据库，可以创建一个中间件或监听器来动态翻译菜单。

### 创建中间件

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Dcat\Admin\Admin;

class SetAdminLocale
{
    public function handle($request, Closure $next)
    {
        if ($user = Admin::user()) {
            // 根据用户的 language 字段设置语言
            $locale = $user->language ?? 'zh_CN';
            app()->setLocale($locale);
        }

        return $next($request);
    }
}
```

### 在 bootstrap/app.php 中注册中间件

```php
->middleware([
    // ...
    \App\Http\Middleware\SetAdminLocale::class,
])
```

## 推荐方案

推荐使用**方法一 + 方法三的组合**：

1. 在语言文件中定义菜单翻译
2. 在数据库中使用翻译键
3. 使用中间件根据用户语言设置动态切换语言

这样可以实现：
- 用户登录后自动切换到其设置的语言
- 菜单标题根据语言自动翻译
- 保留数据库中的翻译键，便于维护

## 实现步骤

1. 添加菜单翻译到语言文件
2. 创建迁移更新菜单标题为翻译键
3. 创建中间件设置用户语言
4. 在用户设置页面添加语言切换功能
