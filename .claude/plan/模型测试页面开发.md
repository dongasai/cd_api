# 模型测试页面开发计划

## 任务概述

在后台创建模型测试页面，支持对渠道->模型进行一次性对话测试，并建立预设系统提示词CRUD管理功能。

## 需求分析

### 功能需求
- **双模式测试**:
  - **渠道直接测试**: 直接使用渠道驱动测试，绕过本系统代理逻辑
  - **系统API测试**: 通过本系统API测试，走完整代理流程（使用系统设置中的默认API Key）
- **流式切换**: 支持流式响应（SSE）和非流式响应两种模式
- **预设提示词**: 提供CRUD管理功能，可快速选择预设提示词和请求头
- **测试记录**: 统一保存两种测试记录，通过test_type字段区分

### 技术栈
- **框架**: Laravel 12.53.0
- **后台面板**: Dcat Admin v2
- **PHP版本**: 8.3.6
- **数据库**: MySQL

## 架构设计

### 数据库设计

#### 表1: `preset_prompts` (预设提示词)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint unsigned | 主键 |
| name | varchar(100) | 提示词名称 |
| category | varchar(50) | 分类（通用、编程、翻译、分析等） |
| content | text | 提示词内容 |
| variables | json | 变量模板 |
| headers | json | 预设HTTP头部信息 |
| is_enabled | tinyint(1) | 是否启用 |
| sort_order | int unsigned | 排序 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

**索引**:
- (category, is_enabled)
- (sort_order)

**字段说明**:
- `headers`: JSON格式存储预设的HTTP头部，例如：
  ```json
  {
    "X-Custom-Auth": "Bearer token123",
    "X-Request-Source": "test-panel",
    "Content-Type": "application/json"
  }
  ```

#### 表2: `model_test_logs` (模型测试记录)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint unsigned | 主键 |
| test_type | enum | 测试类型（channel_direct/system_api） |
| channel_id | bigint unsigned | 渠道ID（channel_direct时有值） |
| channel_name | varchar(100) | 渠道名称（冗余） |
| model | varchar(100) | 测试模型 |
| actual_model | varchar(100) | 实际上游模型 |
| api_key_id | bigint unsigned | API Key ID（system_api时有值） |
| api_key_name | varchar(100) | API Key名称（冗余） |
| prompt_preset_id | bigint unsigned | 关联提示词ID（可为null） |
| system_prompt | text | 系统提示词 |
| user_message | text | 用户消息 |
| assistant_response | longtext | AI响应 |
| request_headers | json | 请求头部（包含预设headers） |
| is_stream | tinyint(1) | 是否流式 |
| response_time_ms | int unsigned | 响应时间（毫秒） |
| first_token_ms | int unsigned | 首token时间 |
| prompt_tokens | int unsigned | 输入token |
| completion_tokens | int unsigned | 输出token |
| total_tokens | int unsigned | 总token |
| status | enum | 状态（success/failed/timeout） |
| error_message | text | 错误信息 |
| metadata | json | 元数据（包含代理路径、重试信息等） |
| created_at | timestamp | 创建时间 |

**索引**:
- (test_type, created_at)
- (channel_id, created_at)
- (api_key_id, created_at)
- (model, created_at)
- (status)

**字段说明**:
- `test_type`: 测试类型
  - `channel_direct`: 直接渠道测试，绕过系统代理
  - `system_api`: 系统API测试，走完整代理流程
- `api_key_id`: 系统API测试时使用的API Key ID
- `request_headers`: 记录实际发送的HTTP头部（包含预设的headers）
- `metadata`: 扩展字段，可存储代理路径、重试次数、匹配规则等信息

## 技术要点

### 系统设置配置
在 `system_settings` 表中添加默认API Key配置：
```php
// 系统设置键名: default_test_api_key
[
    'group' => 'test',
    'key' => 'default_test_api_key',
    'value' => 'sk-xxxx', // API Key值
    'type' => 'string',
    'label' => '默认测试API Key',
    'description' => '用于模型测试功能的默认API Key',
]
```

### 双模式测试流程

#### 模式1: 渠道直接测试
```
用户选择渠道 + 模型
    ↓
直接调用渠道驱动（绕过系统代理）
    ↓
Channel->test() 或 ChannelDriver->test()
    ↓
返回测试结果
```

#### 模式2: 系统API测试
```
用户选择模型（无需选渠道）
    ↓
读取系统设置的默认API Key
    ↓
调用本系统API: /api/openai/v1/chat/completions
    ↓
走完整代理流程（渠道选择、重试、日志等）
    ↓
返回测试结果
```

## 注意事项

1. **安全性**: 验证用户权限，防止未授权访问
2. **超时处理**: 设置合理的请求超时时间

4. **性能优化**: 避免频繁的数据库查询，使用缓存
5. **兼容性**: 确保不支持 SSE 的浏览器有降级方案
