# 安装/升级网页详细执行计划

## 方案概述
- **方案**：简洁向导式安装页面
- **安全**：文件锁机制（storage/installed.lock）
- **风格**：独立简洁风格，不依赖后台框架

---

## 步骤设计

### 安装流程

1. **步骤1：APP_KEY 检测与生成**
   - 检测 `.env` 文件是否存在
   - 不存在则复制 `.env.example` 到 `.env`
   - 检测 `APP_KEY` 是否存在
   - 不存在则自动调用 `php artisan key:generate` 生成
   - APP_KEY 是 Laravel 运行的前提，必须优先处理

2. **步骤2：安装锁检测**
   - 检测 `storage/installed.lock` 文件是否存在
   - 存在则跳转到首页或显示"已安装"提示
   - 不存在则进入下一步

3. **步骤3：环境检测**
   - 检测 PHP 版本（>=8.2）
   - 检测 PHP 扩展（PDO、Mbstring、OpenSSL、Curl）
   - 检测目录权限（storage、bootstrap/cache）
   - 检测 .env 文件状态
   - 全部通过后进入下一步，失败显示修复建议

4. **步骤4：配置填写**
   - 填写 APP_URL
   - 填写数据库配置（连接类型、地址、端口、库名、用户、密码）
   - 测试数据库连接按钮
   - 保存配置到 .env 文件

5. **步骤5：数据库迁移**
   - 执行 `php artisan migrate`
   - 实时显示迁移日志
   - 显示进度百分比
   - 失败时提供重试按钮

6. **步骤6：初始化数据**
   
   **Seeder 执行顺序**：
   1. `AdminTablesSeeder` - 后台菜单、权限、角色（必须先执行，管理员依赖角色）
   2. `SystemSettingSeeder` - 系统配置
   3. `PresetPromptSeeder` - 预设提示词
   4. `ChannelAffinityRuleSeeder` - 渠道亲和性规则
   5. 创建管理员账号并关联 Administrator 角色

7. **步骤7：安装完成**
   - 创建安装标记文件 `storage/installed.lock`
   - 显示成功提示
   - 显示管理员账号摘要
   - 跳转到后台登录页 `/admin`

### 升级流程

1. **检测待执行迁移**
   - 显示当前版本
   - 列出待执行的迁移文件

2. **执行升级**
   - 点击按钮执行迁移
   - 实时显示执行日志
   - 显示进度指示器

3. **完成升级**
   - 显示成功提示
   - 返回后台按钮

## 步骤2：创建安装检查中间件

### 文件
- `laravel/app/Http/Middleware/InstalledMiddleware.php`
- `laravel/bootstrap/app.php`（注册中间件）

### 逻辑
```php
class InstalledMiddleware {
    public function handle($request, $next) {
        $lockFile = storage_path('installed.lock');
        if (file_exists($lockFile)) {
            // 已安装，禁止访问安装页面
            return redirect('/')->with('error', '系统已安装');
        }
        return $next($request);
    }
}
```

| bootstrap/cache目录 | 可写 | 必须 |

### 实现逻辑
```php
class EnvironmentChecker {
    public function check(): array {
        return [
            'php_version' => $this->checkPhpVersion(),
            'extensions' => $this->checkExtensions(),
            'directories' => $this->checkDirectories(),
            'env_file' => $this->checkEnvFile(),
        ];
    }
}

## 步骤5：实现配置填写和保存

### 配置项
| 配置 | 字段名 | 说明 |
|------|--------|------|
| APP_KEY | - | **自动生成**，安装时调用 `php artisan key:generate` |
| APP_URL | app_url | 应用地址 |
| DB_CONNECTION | db_connection | 数据库类型（mysql/sqlite） |
| DB_HOST | db_host | 数据库地址 |
| DB_PORT | db_port | 数据库端口 |
| DB_DATABASE | db_database | 数据库名 |
| DB_USERNAME | db_username | 数据库用户 |
| DB_PASSWORD | db_password | 数据库密码 |

### APP_KEY 自动生成逻辑
```php
// 在保存配置前自动生成 APP_KEY
Artisan::call('key:generate');
// 或手动生成
$key = 'base64:' . base64_encode(random_bytes(32));
```

### 保存逻辑
```php
class ConfigService {
    public function save(array $config): bool {
        // 先生成 APP_KEY
        if (!file_exists(base_path('.env')) || !env('APP_KEY')) {
            Artisan::call('key:generate');
        }
        
        $envContent = $this->generateEnvContent($config);
        file_put_contents(base_path('.env'), $envContent);
        // 设置文件权限
        chmod(base_path('.env'), 0600);
        return true;
    }
    
    public function testDatabaseConnection(array $config): array {
        // 临时配置数据库连接进行测试
        try {
            $connection = DB::connection([
                'driver' => $config['db_connection'],
                'host' => $config['db_host'],
                'port' => $config['db_port'],
                'database' => $config['db_database'],
                'username' => $config['db_username'],
                'password' => $config['db_password'],
            ]);
            $connection->getPdo();
            return ['success' => true, 'message' => '数据库连接成功'];
        } catch (\Exception $e) {
            return ['success' => false, 'message' => $e->getMessage()];
        }
    }
}
```

## 步骤7：实现管理员创建

### 现有 Seeder
- `AdminTablesSeeder` - 后台菜单、权限、角色基础数据（不含管理员）
- `AdminUserSeeder` - 前端 User 模型（非后台管理员）

### 创建逻辑
安装流程：
1. 执行 `AdminTablesSeeder` 初始化后台基础数据
2. 创建 Dcat Admin 管理员账号

```php
use Dcat\Admin\Models\AdminUser;
use Dcat\Admin\Models\Role;
use Dcat\Admin\Models\Administrator;

class AdminService {
    public function initializeAdmin(array $data): AdminUser {
        // 先执行后台基础数据Seeder
        Artisan::call('db:seed', ['--class' => 'AdminTablesSeeder']);

        // 创建管理员账号
        $admin = AdminUser::create([
            'username' => $data['username'],
            'password' => bcrypt($data['password']),
            'name' => $data['name'],
        ]);

        // 关联 Administrator 角色
        $admin->roles()->attach(Role::where('slug', 'administrator')->first()->id);

        return $admin;
    }
}
```

### 密码要求
- 最少8位
- 建议包含大小写字母、数字

## 安全机制总结

### 文件锁机制
1. 安装完成时创建 `storage/installed.lock`
2. 中间件 `InstalledMiddleware` 检查文件存在
3. 存在则禁止访问安装/升级页面
4. 手动删除文件可重新安装

### 其他安全
- CSRF Token保护所有表单
- 配置文件权限设置为600
- 密码强度建议（不强制）
