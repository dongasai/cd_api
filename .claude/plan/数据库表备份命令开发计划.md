# 数据库表备份命令开发计划

## 任务概述

开发一个 Artisan 命令，用于备份指定数据表的数据和结构，支持配置文件管理、通配符匹配、大表分批处理、文件清理策略等功能。

## 技术栈

- **框架**: Laravel 12
- **数据库**: MySQL
- **核心组件**: Schema Builder, Query Builder, Filesystem, Console Output

## 实现方案

采用方案 1：基于 Laravel 原生组件实现，完全可控，性能优化空间大。

---

## 详细实现步骤

### 阶段 1: 配置文件设计

**文件**: `config/backup.php`

**功能**:
- 定义备份表组（groups）
- 配置默认备份选项
- 定义存储路径模板
- 配置清理策略

**配置结构**:
```php
return [
    'groups' => [
        'core' => [
            'tables' => ['users', 'admin_*'],
            'with_structure' => true,
            'compress' => true,
        ],
        'logs' => [
            'tables' => ['audit_logs', 'request_logs'],
            'with_structure' => false,
            'chunk_size' => 10000,
        ],
    ],

    'defaults' => [
        'path' => storage_path('backups/tables'),
        'with_structure' => true,
        'compress' => true,
        'chunk_size' => 5000,
        'retention_count' => 10,
    ],

    'cleanup' => [
        'enabled' => true,
        'retention_count' => 10,
    ],
];
```

### 阶段 2: 核心命令开发

**文件**: `app/Console/Commands/BackupTableCommand.php`

**类结构**:
```
BackupTableCommand extends Command
├── 属性
│   ├── $signature (命令签名)
│   └── $description (命令描述)
├── handle() (主入口)
├── resolveTables() (解析表名，支持通配符)
├── backupTable() (备份单个表)
├── exportStructure() (导出表结构)
├── exportData() (导出数据，支持分批)
├── compressFile() (压缩备份文件)
├── cleanupOldBackups() (清理旧备份)
└── formatBytes() (格式化字节大小)
```

**命令签名**:
```php
signature = 'backup:table
    {--group= : 指定配置文件中的表组}
    {--tables= : 直接指定表名，逗号分隔}
    {--path= : 自定义备份路径}
    {--no-structure : 不包含表结构}
    {--no-compress : 不压缩备份文件}
    {--chunk= : 自定义分批大小}
    {--keep= : 保留最近N个备份}'
```

### 阶段 3: 表名解析服务

**文件**: `app/Services/Backup/TableResolver.php`

**职责**:
- 解析配置中的表组
- 支持通配符匹配（`admin_*`）
- 支持正则表达式匹配
- 验证表是否存在

**方法**:
```php
class TableResolver
{
    public function resolveGroup(string $group): array
    public function resolvePattern(string $pattern): array
    public function validateTables(array $tables): array
    private function matchByWildcard(string $pattern): array
    private function matchByRegex(string $pattern): array
}
```

---

### 阶段 4: 数据导出器

**文件**: `app/Services/Backup/DataExporter.php`

**职责**:
- 导出表结构（SHOW CREATE TABLE）
- 导出数据（SELECT + INSERT）
- 支持分批查询避免内存溢出
- 生成 SQL 文件格式

**方法**:
```php
class DataExporter
{
    public function exportStructure(string $table): string
    public function exportData(string $table, int $chunkSize): string
    private function generateInsertStatements(string $table, int $offset, int $limit): string
    private function escapeValue($value): string
}
```

### 阶段 5: 文件管理器

class BackupFileManager

## 预期成果

### 使用示例

```bash
# 备份配置文件中的 core 表组
php artisan backup:table --group=core

# 备份指定表
php artisan backup:table --tables=users,channels

# 备份所有 admin_ 开头的表
php artisan backup:table --tables=admin_*

# 自定义路径和保留数量
php artisan backup:table --group=logs --path=/tmp/backups --keep=20

# 只备份数据，不包含结构
php artisan backup:table --tables=audit_logs --no-structure

# 定时任务（在 app/Console/Kernel.php 中）
$schedule->command('backup:table --group=core')->daily()->at('02:00');
$schedule->command('backup:table --group=logs')->daily()->at('03:00');
```

### 输出文件示例

```
storage/backups/tables/
├── users_20260317_234500.sql.gz
├── channels_20260317_234500.sql.gz
├── audit_logs_20260317_234500.sql.gz
└── ...
```

## 性能考虑

### 大表处理策略

1. **分批查询**: 使用 `chunk()` 方法，默认每批 5000 行
2. **内存管理**: 及时写入文件，不积累在内存
3. **进度显示**: 使用 Symfony Console Progress Bar
4. **超时处理**: 设置合适的 PHP 超时时间

### 示例数据量评估

- `audit_logs`: 预估百万级，需要分批处理
- `channel_request_logs`: 预估十万级
- `users`: 预估千级

---

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 大表内存溢出 | 高 | 强制分批处理，限制单次查询行数 |
| 表不存在 | 中 | 执行前验证表名，跳过不存在的表 |
| 磁盘空间不足 | 高 | 备份前检查空间，清理策略生效 |
| 权限问题 | 中 | 检查存储目录权限，给出明确错误提示 |
| 长时间锁定表 | 高 | 使用事务 + 非锁定读取 |
