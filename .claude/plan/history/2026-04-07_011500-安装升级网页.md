# 任务：创建安装/升级网页

## 任务描述
创建安装/升级网页，通过网页进行安装/升级，升级主要是数据库，代码不通过网页升级，需要先手动更新代码

### 确定的需求

**安装流程（4步骤）**：
1. 环境检测 - PHP版本、数据库连接、目录权限
2. 安装状态检测 - 已安装则跳转
3. 配置填写 - 数据库配置、APP_URL等
4. 数据库迁移 - 执行迁移
5. 创建管理员账号

**升级流程**：
- 手动触发访问升级页面
- 执行待运行的迁移文件
- 显示执行结果

**页面风格**：独立风格（简洁安装向导）

## 头脑风暴汇总

### 关键创意方向

1. **安装向导交互**：分步进度条、步骤导航、进度百分比
2. **环境检测**：分组展示（PHP扩展、目录权限、数据库），状态图标
3. **配置填写**：实时验证、智能默认值、密码强度指示器
4. **安装标记**：双标记机制（文件+数据库）
5. **升级页面**：迁移列表、实时日志、执行按钮
6. **错误处理**：分类展示、一键复制、断点恢复
7. **安全性**：安装后禁用路由、CSRF保护、IP限制

## 构思阶段 - 方案收敛

### 方案 A：简洁向导式（推荐）

**安装流程**：环境检测 → 配置填写 → 数据库迁移 → 创建管理员 → 完成

**关键特性**：
- 顶部水平进度条，可回溯已完成步骤
- 环境检测分组展示（PHP扩展、目录权限、数据库连接）
- 配置填写分组卡片，实时验证
- 双标记机制（storage/installed + 数据库 installation 表）
- 升级页面：迁移列表 + 实时日志
- 错误分类、一键复制、分步恢复
- 安装后禁用路由、CSRF保护

### 方案 B：终端风格（技术感强）

**核心理念**：模拟终端输出，实时日志显示

**关键特性**：
- 黑底白字终端界面
- 实时日志推送（SSE/WebSocket）
- 圆形进度环
- 一键智能安装

### 推荐：方案 A

理由：用户覆盖面广、实现复杂度适中、错误处理友好、移动端适配良好
