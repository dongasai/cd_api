# 安装流程前端验证修复

## 问题

安装流程中前端验证未生效，导致请求直接到达后端触发 ValidationException。

## 原因

前端使用 `e.preventDefault()` 阻止表单默认提交行为，导致 HTML5 表单验证（minlength、required等）未触发。

## 解决方案

在 JavaScript 提交处理中，先调用 `form.checkValidity()` 和 `form.reportValidity()` 进行前端验证，验证成功后才阻止默认行为并执行 AJAX 提交。

### 修改文件

1. **laravel/resources/views/install/admin.blade.php**
   - 添加 `this.checkValidity()` 和 `this.reportValidity()` 验证检查
   - 添加验证错误处理（`response.errors`）

2. **laravel/resources/views/install/config.blade.php**
   - 同样添加前端验证检查
   - 添加验证错误处理

3. **laravel/lang/zh_CN/validation.php**
   - 新增中文验证消息翻译文件
   - 解决 `validation.min.string` 等翻译键显示问题

4. **laravel/bootstrap/app.php**
   - 添加 ValidationException 的异常处理
   - 返回 JSON 格式的验证错误响应

## 验证规则一致性

| 字段 | 前端验证 | 后端验证 |
|------|---------|---------|
| username | required, minlength="3", maxlength="50" | required, min:3, max:50 |
| password | required, minlength="8" | required, min:8 |
| name | required, maxlength="100" | required, max:100 |
