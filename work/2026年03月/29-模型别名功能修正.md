# 模型别名功能修正工作日志

**修正时间**：2026-03-29 02:50

## 问题发现

用户测试发现别名功能未生效，错误提示：
```
状态码：400
错误：No available channel for model 'glm-5' with current API key restrictions
```

**根本原因**：
- 原实现采用"降级策略"
- 只在找不到渠道时才尝试别名
- API Key 限制过滤导致无渠道时，不触发别名逻辑

## 用户反馈

**关键洞察**：
> 别名应该是同级别的，不是降级。正确流程是：入参 → Key映射 → 所有别名 → 搜索渠道

这个观点完全正确！别名是**同级扩展**，不是降级。

## 解决方案

### 策略调整

**错误流程（旧）**：
```
请求 glm-5
   ↓
搜索 glm-5 渠道 → 失败
   ↓
尝试 GLM-5 渠道（降级）
```

**正确流程（新）**：
```
请求 glm-5
   ↓
获取所有名称：['glm-5', 'GLM-5']
   ↓
搜索所有名称的渠道：[2, 3, 5, 7]
   ↓
过滤与负载均衡
```

### 代码修改

**文件**：[app/Services/Router/ChannelRouterService.php](app/Services/Router/ChannelRouterService.php)

**新增方法**：
1. `getModelNamesWithAliases(string $model): array` - 获取模型名+别名
2. `getAvailableChannelsForModels(array $modelNames): Collection` - 多模型名称搜索渠道

**修改逻辑**：
- `selectChannel()` 开始就扩展别名
- 一次性搜索所有名称对应的渠道
- 删除所有"降级"相关代码

**删除方法**：
- `tryAliasFallback()` - 不再需要降级逻辑
