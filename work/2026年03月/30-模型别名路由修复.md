# 模型别名路由修复

## 问题现象

用户请求模型 `claude-haiku-4-5-20251001`，路由到 channel_id=7（阿里coding-Openai），但上游 API 返回错误：
```
model `claude-haiku-4-5-20251001` is not supported.
```

## 问题分析

### 配置情况

1. **模型别名配置**：
   - `claude-haiku-4-5-20251001` 的别名：`["minimax-m2.5", "minimax-m2.1", "stepfun-ai/Step-3.5-Flash", "MiniMax-M2.5"]`

2. **渠道模型映射**：
   - channel_id=7 有配置 `MiniMax-M2.5` → `MiniMax-M2.5`
   - 但**没有** `claude-haiku-4-5-20251001` 的映射

### 原有逻辑流程

1. 别名扩展用于**渠道选择**：通过别名找到支持该模型的渠道
2. 模型映射 `resolveModel()` 只做**精确匹配**：查找 `model_name = 'claude-haiku-4-5-20251001'`
3. 精确匹配失败，返回原模型名
4. 发送给上游的仍是 `claude-haiku-4-5-20251001`，导致错误

## 解决方案

### 核心思路

**当通过别名匹配到渠道后，使用匹配成功的别名（及其映射）作为实际请求模型**

### 实现修改

修改 [ChannelRouterService.php](../laravel/app/Services/Router/ChannelRouterService.php) 的 `resolveModel()` 方法：

```php
public function resolveModel(string $model, ?Channel $channel = null): string
{
    if ($channel) {
        // 优先查找：通过别名匹配找到该渠道，则使用匹配别名对应的映射模型
        $modelNames = \App\Services\ModelService::resolveModelWithAliases($model);
        foreach ($modelNames as $modelName) {
            // 检查该别名是否在该渠道的 channel_models 表中
            $channelModel = \App\Models\ChannelModel::where('channel_id', $channel->id)
                ->where('model_name', $modelName)
                ->where('is_enabled', true)
                ->first();

            if ($channelModel) {
                // 找到匹配别名，使用映射后的模型名（如果没有映射则使用别名本身）
                $resolvedModel = $channelModel->mapped_model ?? $modelName;
                return $resolvedModel;
            }
        }
    }

    return $model;
}
```

### 工作流程

1. 获取模型的所有别名：`resolveModelWithAliases('claude-haiku-4-5-20251001')`
   → 返回：`["claude-haiku-4-5-20251001", "minimax-m2.5", "minimax-m2.1", "stepfun-ai/Step-3.5-Flash", "MiniMax-M2.5"]`

2. 遍历每个别名，在 channel_id=7 的 `channel_models` 表中查找：
   - `claude-haiku-4-5-20251001` → 找到映射记录（之前添加的）
   - 其他别名 → 未找到

3. 使用找到的映射：`MiniMax-M2.5`

4. 发送给上游的模型名：`MiniMax-M2.5`

模型名成功替换为 `MiniMax-M2.5`，不再出现 "model not supported" 错误。

## 意义

- **优化别名机制**：别名不仅用于渠道选择，还用于模型映射
- **减少配置工作**：不需要为每个渠道都配置所有模型映射，只需配置别名对应的映射即可
- **提升系统灵活性**：支持通过别名自动适配不同渠道的模型命名差异
