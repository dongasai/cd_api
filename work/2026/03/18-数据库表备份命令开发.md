# 数据库表备份命令开发

**开发时间**: 2026年3月18日
**开发人员**: Claude AI

## 任务概述

开发一个 Artisan 命令，用于备份指定数据表的数据和结构。

## 需求说明

根据用户需求，备份命令需要支持以下特性：

1. **表指定方式**: 配置文件预定义表组 + 支持通配符/正则匹配
2. **备份内容**: 数据 + 表结构（可配置）
3. **存储位置**: 自定义路径参数（可配置）
4. **大表处理**: 分批导出避免内存溢出
5. **备份策略**: 全量备份，支持定时任务
6. **清理策略**: 保留最近 N 个备份文件
7. **配置文件**: 在 config 中新增备份配置

## 技术方案

### 架构设计

采用服务分层架构，将功能拆分为三个核心服务：

```
BackupTableCommand (命令层)
    ├── TableResolver (表名解析服务)
    ├── DataExporter (数据导出服务)
    └── BackupFileManager (文件管理服务)
```

### 核心组件

#### 1. 配置文件 (config/backup.php)

提供灵活的配置选项：

- **表组定义**: 预配置常用的表组（core, audit, logs, admin, coding）
- **默认配置**: 路径、压缩、分批大小、保留数量等
- **清理策略**: 自动清理旧备份
- **性能优化**: 内存限制、查询超时等
- **排除表**: 缓存、会话等临时表自动排除

#### 2. TableResolver 服务

负责解析表名，支持：

- 精确匹配: `users`
- 通配符匹配: `admin_*`, `coding_*`
- 正则表达式: `/^log_/`
- 表组解析: `--group=core`
- 排除表过滤

**关键实现**:
- 通配符转正则表达式时使用占位符避免转义问题
- 支持 SQLite 和 MySQL 两种数据库驱动
- 自动过滤缓存、会话等临时表

#### 3. DataExporter 服务

负责导出表结构和数据：

- 表结构导出: `SHOW CREATE TABLE` (MySQL) / `sqlite_master` (SQLite)
- 数据导出: 分批查询，支持进度回调
- SQL 格式化: NULL、布尔值、字符串转义
- 统计信息: 行数、数据大小

**关键实现**:
- 使用 chunk 分批处理大表
- 支持自定义分批大小（默认 5000）
- 提供进度回调函数支持实时显示

#### 4. BackupFileManager 服务

负责文件管理：

- 文件名生成: 支持占位符 (`{table}`, `{date}`, `{time}`)
- 文件压缩: gzip 压缩（可选）
- 文件清理: 保留最近 N 个备份
- 工具方法: 格式化字节大小、检查磁盘空间

#### 5. BackupTableCommand 命令

主命令，整合所有服务：

```bash
php artisan backup:table [options]
```

**参数说明**:
- `--group=GROUP`: 指定配置文件中的表组
- `--tables=TABLES`: 直接指定表名，逗号分隔或通配符
- `--path=PATH`: 自定义备份路径
- `--no-structure`: 不包含表结构
- `--no-compress`: 不压缩备份文件
- `--chunk=CHUNK`: 自定义分批大小
- `--keep=KEEP`: 保留最近 N 个备份

## 使用示例

### 1. 备份单个表

```bash
php artisan backup:table --tables=users
```

### 2. 备份多个表

```bash
php artisan backup:table --tables=users,channels,api_keys
```

### 3. 使用通配符

```bash
php artisan backup:table --tables=admin_*
php artisan backup:table --tables=coding_*
```

### 4. 使用表组

```bash
php artisan backup:table --group=core
php artisan backup:table --group=audit
```

### 5. 自定义选项

```bash
# 不压缩
php artisan backup:table --tables=users --no-compress

# 仅备份数据（不包含结构）
php artisan backup:table --tables=users --no-structure

# 自定义路径
php artisan backup:table --tables=users --path=/tmp/backups

# 保留最近 5 个备份
php artisan backup:table --tables=users --keep=5

# 自定义分批大小
php artisan backup:table --tables=large_table --chunk=1000
```

### 6. 定时任务

在 `app/Console/Kernel.php` 中配置：

```php
protected function schedule(Schedule $schedule)
{
    // 每天凌晨 2 点备份核心表
    $schedule->command('backup:table --group=core')
        ->dailyAt('02:00');

    // 每周备份审计日志
    $schedule->command('backup:table --group=audit')
        ->weekly();
}
```

## 技术亮点

### 1. 数据库兼容性

支持 SQLite 和 MySQL 两种数据库驱动，自动适配 SQL 语法：

```php
// 获取表结构
if ($driver === 'sqlite') {
    $result = DB::select("SELECT sql FROM sqlite_master WHERE type='table'");
} else {
    $result = DB::select("SHOW CREATE TABLE `{$table}`");
}
```

### 2. 通配符转义

使用占位符避免正则表达式转义问题：

```php
$placeholder1 = uniqid('WILDCARD_STAR_');
$temp = str_replace('*', $placeholder1, $pattern);
$regex = preg_quote($temp, '/');
$regex = str_replace($placeholder1, '.*', $regex);
```

### 3. 大表优化

使用 chunk 分批处理，避免内存溢出：

```php
DB::table($table)
    ->orderBy('id')
    ->chunk($chunkSize, function ($rows) {
        // 处理每批数据
    });
```

### 4. 进度反馈

实时显示备份进度：

```php
$exporter->exportData($table, $chunkSize, function ($processed, $total) {
    $percent = round(($processed / $total) * 100, 1);
    $this->info("  进度: {$processed}/{$total} ({$percent}%)");
});
```

### 5. 自动清理

备份完成后自动清理旧文件，保持目录整洁：

```php
// 按修改时间排序（新到旧）
usort($files, function ($a, $b) {
    return filemtime($b) - filemtime($a);
});

// 删除超出保留数量的文件
$toDelete = array_slice($files, $keep);
```

## 性能考虑

1. **内存使用**: 分批处理，每批默认 5000 行
2. **磁盘空间**: 支持压缩，节省存储空间
3. **查询性能**: 使用 ORDER BY id 确保分批一致性
4. **文件大小**: 压缩率约 50%（SQL → gzip）
