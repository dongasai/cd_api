cd laravel && php artisan db:seed --class=PresetPromptSeeder

### 模型选择优化

#### 功能改进
将模型输入改为下拉选择，并实现与渠道下拉的联动功能。

#### 实现细节
1. **双输入方式**：
   - 渠道直接测试：使用下拉选择框，自动加载渠道支持的模型
   - 系统API测试：使用文本输入框，允许手动输入任意模型名称

2. **联动逻辑**：
   - 选择测试类型为"渠道直接测试"时，显示下拉框
   - 选择渠道后，通过 AJAX 自动加载该渠道支持的模型列表
   - 选择测试类型为"系统API测试"时，切换为文本输入框

3. **用户体验优化**：
   - 显示加载状态提示
   - 显示已加载模型数量
   - 加载失败时的错误提示

#### 技术要点
- 使用 `display: none` 切换显示不同的输入元素
- 根据测试类型动态获取正确的模型值
- AJAX 异步加载模型列表
- 响应式更新下拉选项

- `laravel/resources/views/admin/model-test/index.blade.php`

### 错误修复

#### 问题描述
模型测试页面直接显示 JavaScript 源代码，浏览器控制台报错：
```
Uncaught SyntaxError: Failed to execute 'appendChild' on 'Node': Invalid or unexpected token
```

#### 根本原因
控制器中使用 `view()->render()` 将视图渲染成字符串，然后传递给 Card 组件。Card 组件在渲染时会对内容进行 HTML 转义，导致 JavaScript 代码中的特殊字符（如 `<`, `>`）被转义成 `&lt;`, `&gt;`，从而引发语法错误。

#### 解决方案
彻底重写测试页面，使用自定义布局而不是 Card 组件：

1. 创建新的视图文件 `index.blade.php`，直接输出内容不继承模板
2. 使用 Bootstrap 的 box 组件构建页面布局，左右两栏分布
3. 所有 JavaScript 代码直接写在视图文件中，避免转义问题
4. 修改控制器，直接返回视图，不再使用 Card 和 Row 组件

#### 修改文件
- 新建：`laravel/resources/views/admin/model-test/index.blade.php`
- 修改：`laravel/app/Admin/Controllers/ModelTestController.php`
- 删除：`laravel/resources/views/admin/model-test/form.blade.php`
- 删除：`laravel/resources/views/admin/model-test/result.blade.php`

#### 技术要点
1. Dcat Admin 视图直接输出内容，不需要模板继承
2. JavaScript 字符串拼接：使用普通字符串拼接而非 ES6 模板字符串
3. Dcat Admin 布局：使用 Bootstrap 的 box 组件
4. 前端交互：jQuery + AJAX + EventSource (SSE)

#### 测试结果
页面正常显示，无 JavaScript 错误，表单交互正常。
