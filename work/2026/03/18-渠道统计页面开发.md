# 渠道统计页面开发

## 时间
2026-03-18

## 背景
用户需要一个专门的渠道统计页面，用于深入分析各个渠道的使用情况。当前首页仪表盘只提供了简单的渠道分布饼图（今日数据），无法满足按渠道深度分析、历史趋势查看、客户端分布、性能指标分析等需求。

## 实现内容

### 1. 创建控制器
创建了 `ChannelStatsController` 控制器，实现以下功能：

#### 基础统计卡片
- **总请求数卡片** - 显示指定时间范围的总请求数及环比增长率
- **总Token卡片** - 显示总Token数及环比增长率
- **总费用卡片** - 显示总费用及环比增长率
- **成功率卡片** - 显示请求成功率及环比增长率

#### 性能指标卡片
- **平均延迟卡片** - 显示平均响应延迟（毫秒）及环比增长
- **首Token延迟卡片** - 显示平均首Token时间（毫秒）及环比增长
- **成功率趋势图** - 显示成功率的迷你趋势图

#### 图表组件
- **请求/Token趋势图** - 双Y轴折线图展示每日请求量和Token数的变化趋势
- **客户端分布饼图** - 按User-Agent前缀分组展示Top 10客户端分布
- **Top User-Agent表格** - 显示请求数最多的10个User-Agent详情

#### 筛选功能
- **渠道选择器** - 支持选择单个渠道或查看所有渠道的汇总数据
- **时间范围选择** - 支持今天、最近7天、最近30天、自定义日期范围

### 2. 添加路由
在 `laravel/app/Admin/routes.php` 中添加了渠道统计页面的路由：
```php
$router->get('channel-stats', 'ChannelStatsController@index')->name('channel-stats');
```

### 3. 添加菜单项
- 在数据库 `admin_menu` 表中添加了菜单项
- 在语言文件中添加了中英文翻译：
  - 中文：渠道统计
  - 英文：Channel Statistics

### 4. 性能优化考虑
- 利用现有索引：`channel_id_created_at_index` 复合索引
- 在数据库层面完成聚合计算（COUNT、SUM、AVG、GROUP BY）
- 只查询必要字段，避免 SELECT *
- 填充缺失日期确保时间轴连续

## 数据源

使用 `audit_logs` 表作为主数据源，包含以下关键字段：
- 渠道信息：`channel_id`, `channel_name`
- Token和费用：`total_tokens`, `cost`
- 性能指标：`latency_ms`, `first_token_ms`
- 状态码：`status_code`（用于计算成功率）
- 客户端信息：`user_agent`, `client_ip`
- 时间戳：`created_at`

## 技术要点

### 环比增长率计算
```php
// 当前周期数据
$currentData = AuditLog::whereBetween('created_at', [$start, $end])->count();

// 上一周期数据（用于环比计算）
$previousStart = (clone $start)->subDays($end->diffInDays($start) + 1);
$previousEnd = (clone $start)->subSecond();
$previousData = AuditLog::whereBetween('created_at', [$previousStart, $previousEnd])->count();

// 环比增长率
$growthRate = $previousData > 0
    ? round(($currentData - $previousData) / $previousData * 100, 1)
    : ($currentData > 0 ? 100 : 0);
```

### 性能指标的特殊处理
对于延迟类的指标，数值下降是好事，因此：
- 增长率为负时显示绿色向上箭头（表示性能改善）
- 增长率为正时显示红色向下箭头（表示性能下降）

### 日期填充
为确保趋势图的时间轴连续，使用以下算法填充缺失日期：
```php
// 生成完整日期范围
$dateRange = [];
$current = clone $start;
while ($current <= $end) {
    $dateRange[$current->format('Y-m-d')] = [
        'date' => $current->format('Y-m-d'),
        'request_count' => 0,
        'token_count' => 0,
    ];
    $current->addDay();
}

// 填充实际数据
foreach ($dailyStats as $stat) {
    $dateRange[$stat->date] = [
        'date' => $stat->date,
        'request_count' => $stat->request_count,
        'token_count' => $stat->token_count,
    ];
}
```

- 简单卡片显示当前成功率和与上一小时的环比数据

### 2026-03-18 23:00 - 增加 User-Agent 分组统计
**需求**: 增加 "Top 10 User-Agent 分组版本" 统计，按用户代理类型分组显示。

**实现**:
1. 新增 `buildTopUserAgentGroupsTable()` 方法构建分组统计表格
2. 新增 `groupUserAgent()` 方法实现 User-Agent 分组逻辑

**分组规则**:
- `claude-cli` - Claude CLI 工具
- `RooCode` - RooCode 工具
- `curl` - curl 命令行工具
- `Chrome` - Chrome 浏览器
- `Firefox` - Firefox 浏览器
- `Safari` - Safari 浏览器
- `Python` - Python 相关工具（requests等）
- `Node.js` - Node.js / axios
- `Go` - Go HTTP 客户端
- `Java` - Java 客户端
- `OpenAI-SDK` - OpenAI SDK
- `Anthropic-SDK` - Anthropic SDK
- `Other` - 其他未识别的 User-Agent

**统计维度**:
- User-Agent 分组名称
- 该分组包含的版本数量
- 总请求数
- 总Token数
- 总费用
- 平均延迟
- 首Token平均延迟

**优势**:
- 更清晰地了解客户端类型分布
- 避免同一工具的不同版本被分开统计
- 提供性能指标对比，便于发现不同客户端的性能差异

### 2026-03-18 22:46 - 修复请求/Token趋势图无法显示问题
**问题**: 趋势图无法显示，浏览器控制台报错 `TypeError: f.yLbFormatter is not a function`。

**原因**:
- ApexCharts 无法正确解析字符串形式的 JavaScript formatter 函数
- 错误配置示例: `'formatter' => 'function(val) { return val.toLocaleString(); }'`

**解决方案**:
- 移除所有字符串形式的 formatter 配置
- 使用 ApexCharts 默认的数字格式化
- 涉及方法: `buildTrendChart()`, `buildSuccessRateTrendCard()`, `buildClientDistributionChart()`

**修改位置**:
- `buildTrendChart()`: 移除 yaxis 的 labels.formatter
- `buildSuccessRateTrendCard()`: 移除 yaxis 的 labels.formatter
- `buildClientDistributionChart()`: 移除 dataLabels 的 formatter
