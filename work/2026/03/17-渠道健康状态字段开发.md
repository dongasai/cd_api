# 渠道健康状态字段开发

**日期**: 2026年03月17日
**类型**: 功能开发

## 需求概述

为渠道表增加健康状态字段，用于标识渠道是否可以参与渠道路由选择。

### 字段说明

1. **status2** - 健康状态字段
   - 类型: `enum('normal', 'disabled')`
   - 默认值: `normal`
   - 说明: 标识渠道的健康状态

2. **status2_remark** - 健康状态备注
   - 类型: `text`, 可空
   - 说明: 记录健康状态变更的原因，例如 "CodingStatus禁用"

### 业务规则

- 当 `status2 = 'disabled'` 时，该渠道不参与渠道路由选择
- `status2_remark` 用于记录禁用原因，便于运维排查

## 问题修复

### 错误：Undefined property: App\Admin\Controllers\ChannelController::$status2

**原因**: 在列表页的健康状态列中，`help()` 方法的闭包无法正确访问 `$this`。

**解决方案**: 改用 `display()` 方法来显示健康状态和备注信息。

修改前：
```php
$grid->column('status2', '健康状态')
    ->using([...])
    ->label([...])
    ->help(function () {
        if ($this->status2 === 'disabled' && $this->status2_remark) {
            return $this->status2_remark;
        }
        return '';
    });
```

修改后：
```php
$grid->column('status2', '健康状态')->display(function ($value) {
    $labels = [
        'normal' => '<span class="label label-success">正常</span>',
        'disabled' => '<span class="label label-danger">禁用</span>',
    ];

    $label = $labels[$value] ?? $value;

    // 如果禁用且有备注，显示备注
    if ($value === 'disabled' && $this->status2_remark) {
        $label .= ' <small class="text-muted">('.$this->status2_remark.')</small>';
    }

    return $label;
});
```

### 错误：Method Illuminate\Support\Str::unescape does not exist

**原因**: `unescape()` 方法在 Dcat Admin 的 Grid\Column 中不存在。

**解决方案**: 移除 `unescape()` 方法调用，直接在 `display()` 中返回 HTML 内容即可。
