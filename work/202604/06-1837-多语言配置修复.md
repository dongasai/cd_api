# 多语言配置修复

## 问题描述

后台管理界面显示翻译键而非翻译文本，如 `admin-actions.reset_api_key`。

## 问题分析

1. **`.env` 语言配置错误**: 设置为 `APP_LOCALE=en` 而非 `zh_CN`
2. **语言路径配置问题**: Laravel 默认使用 `resources/lang` 目录，而项目的语言文件位于 `lang/` 目录

## 解决方案

### 1. 修复 .env 语言配置

```diff
- APP_LOCALE=en
- APP_FALLBACK_LOCALE=en
- APP_FAKER_LOCALE=en_US
+ APP_LOCALE=zh_CN
+ APP_FALLBACK_LOCALE=zh_CN
+ APP_FAKER_LOCALE=zh_CN
```

### 2. 设置语言文件路径

在 `app/Providers/AppServiceProvider.php` 的 boot 方法中添加：

```php
public function boot(): void
{
    // 设置语言文件路径为 lang 目录
    $this->app->useLangPath(base_path('lang'));
}
```

### 3. 合并语言文件

将 `resources/lang/zh_CN/` 目录下的新增语言文件（`admin-mcp-client.php`、`admin-search-driver.php`）合并到 `lang/zh_CN/` 目录。

### 4. 创建英文语言文件

为新增的功能创建对应的英文语言文件：
- `lang/en/admin-mcp-client.php`
- `lang/en/admin-search-driver.php`

## 涉及文件

- `.env` - 语言配置
- `app/Providers/AppServiceProvider.php` - 语言路径设置
- `lang/zh_CN/*.php` - 中文语言文件
- `lang/en/*.php` - 英文语言文件

## 验证结果

```bash
$ php artisan tinker --execute="echo trans('admin-actions.reset_api_key');"
重置密钥  # 正确显示中文翻译
```

## 后续建议

1. 新增语言文件时统一放在 `lang/{locale}/` 目录下
2. 使用 `admin_trans_action()` 等辅助函数时确保对应的语言文件已创建
3. 创建新功能时同时创建中英文双语语言文件
