# 修复生产容器构建问题

**时间**: 2026-04-06 12:31

## 问题

`cd_api_prod` 容器运行异常：
- HTTP 返回 500 错误
- queue-worker、schedule-worker 进入 FATAL 状态
- vendor 目录不存在

## 根因

1. Dockerfile 中 `composer install` 在 php 用户下执行，权限问题导致失败
2. supervisord 配置中 `composer run build-cache` 命令不存在

## 修复

### Dockerfile
- composer install 在 root 用户下执行
- 执行后 chown -R php:php /var/www/html/vendor

### supervisord.conf  
- 移除无效的 build-cache 任务

## 结果

容器重启后所有服务正常运行。
