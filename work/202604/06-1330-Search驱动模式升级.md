# Search 驱动模式升级

> 2026-04-06 13:30

## 背景

将 SearchTool 从单一模拟数据实现升级为可扩展的驱动模式架构，支持多种搜索引擎。

## 实现方案

### 架构设计

参考项目现有的 Protocol Driver 模式，设计 Search 驱动体系：

```
SearchDriverInterface → AbstractSearchDriver → 具体驱动实现
                            ↓
                    SearchDriverManager（管理器）
```

### 新增文件

**Contracts（契约）**:
- `SearchRequest.php` - 搜索请求结构体
- `SearchResult.php` - 搜索结果结构体
- `SearchItem.php` - 搜索结果项

**Driver（驱动）**:
- `SearchDriverInterface.php` - 驱动接口
- `AbstractSearchDriver.php` - 抽象基类，提供通用方法
- `MockSearchDriver.php` - Mock 驱动（模拟数据，测试用）
- `SerperSearchDriver.php` - Serper 驱动（Google 搜索 API）
- `DuckDuckGoSearchDriver.php` - DuckDuckGo 驱动

**管理器**:
- `SearchDriverManager.php` - 驱动管理器
- `SearchDriverException.php` - 异常类

**配置**:
- `config/search.php` - 搜索配置文件

**服务提供者**:
- `SearchServiceProvider.php` - 注册服务

### 修改文件

- `app/Mcp/Tools/SearchTool.php` - 重写为驱动模式
- `bootstrap/providers.php` - 注册 SearchServiceProvider
- `docs/MCP.md` - 更新文档

## 支持的驱动

| 驱动 | 说明 | 配置 |
|------|------|------|
| mock | 模拟数据，测试用 | 无需配置 |
| serper | Google 搜索 (serper.dev) | 需要 `SERPER_API_KEY` |
| duckduckgo | DuckDuckGo 搜索 | 无需配置 |

## 配置使用

```env
# 默认驱动
SEARCH_DRIVER=mock

# Serper API Key（可选）
SERPER_API_KEY=your-key
```

## 新增参数

SearchTool 新增 `driver` 参数，允许调用时指定使用的搜索引擎：
```json
{
  "search_query": "API",
  "driver": "serper"
}
```

## 扩展方式

添加新驱动只需：
1. 创建驱动类继承 `AbstractSearchDriver`
2. 实现 `search()` 方法
3. 在 `config/search.php` 注册

# 数据库表新增

> 2026-04-06 15:40

## 新增表

### search_drivers（搜索驱动配置表）

存储搜索驱动的配置信息，支持动态管理。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| name | varchar(50) | 驱动名称 |
| slug | varchar(50) | 驱动标识（唯一） |
| driver_class | varchar(200) | 驱动类名 |
| config | json | 驱动配置 |
| timeout | int | 请求超时秒数 |
| priority | int | 优先级 |
| is_default | bool | 是否为默认驱动 |
| status | enum | 状态：active/inactive/error |
| usage_count | bigint | 使用次数 |
| last_used_at | timestamp | 最后使用时间 |

### search_logs（搜索记录表）

记录所有搜索请求，用于统计和分析。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| query | varchar(500) | 搜索查询内容 |
| driver | varchar(50) | 使用的驱动 |
| driver_id | bigint | 驱动ID |
| result_count | int | 返回结果数量 |
| total_count | int | 总匹配数量 |
| success | bool | 是否成功 |
| response_time_ms | int | 响应时间(毫秒) |
| filters | json | 过滤条件 |
| results | json | 搜索结果摘要 |
| client_ip | varchar(45) | 客户端IP |

## 新增模型

- `App\Models\SearchDriver` - 搜索驱动配置模型
- `App\Models\SearchLog` - 搜索记录模型

## 初始数据

已插入三个默认驱动：
1. Mock 搜索（默认，活跃）
2. Serper (Google)（需配置 API Key，未激活）
3. DuckDuckGo（活跃）

## 驱动管理器升级

`SearchDriverManager` 现在优先从数据库读取驱动配置：
- 数据库不可用时自动回退到配置文件
- 支持驱动使用统计
- 支持驱动错误标记

## 搜索日志记录

`SearchTool` 自动记录每次搜索：
- 成功搜索记录结果摘要
- 失败搜索记录错误信息
- 记录响应时间用于性能分析
