# MCP 客户端功能升级

## 时间
2026-04-06 17:42

## 背景
原有的 `php-mcp/client` 库不支持 Streamable HTTP 传输模式，无法连接使用 `laravel/mcp` 搭建的 MCP 服务器。

## 问题分析
1. `php-mcp/client` 使用 HTTP+SSE 模式（GET 请求建立 SSE 连接）
2. `laravel/mcp` 仅支持 Streamable HTTP 模式（POST 请求，单端点）
3. 两者不兼容，导致 HTTP 405 错误

## 解决方案
切换到官方 `mcp/sdk` (dev-main 版本)，该 SDK 支持 Streamable HTTP 传输。

## 完成工作

### 1. 安装 mcp/sdk
```bash
composer require mcp/sdk:dev-main
```

### 2. 重写 McpClientService
- 使用 `Mcp\Client` 替代 `PhpMcp\Client\Client`
- 使用 `Mcp\Client\Transport\HttpTransport` 处理 HTTP 传输
- 使用 `Mcp\Client\Transport\StdioTransport` 处理 Stdio 传输
- 更新 API 适配新的结果类型

### 3. 创建测试命令
```bash
php artisan cdapi:mcp:test --url=<MCP_URL> --header=<HEADER>
```

### 4. 修复 Bug
修复 `SearchTool` 基类引用错误：
```php
// 错误
class SearchTool extends Tool

// 正确
use Laravel\Mcp\Server\Tool;
class SearchTool extends Tool
```
