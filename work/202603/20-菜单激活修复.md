# 修复 dcat-admin 子路径菜单激活问题

## 问题描述

访问 `admin/user-agents/create` 页面时，左侧菜单中的 `user_agents` 菜单项无法激活（不高亮）。

## 问题原因

dcat-admin 的 `Menu::isActive()` 方法使用**精确匹配**来判断菜单是否激活：

```php
// 原代码
return trim($this->getPath($item['uri']), '/') == $path;
```

这导致：
- 菜单 URI: `user-agents` → 解析为 `admin/user-agents`
- 当前路径: `admin/user-agents/create`
- 精确比较: `admin/user-agents` ≠ `admin/user-agents/create` → **菜单不激活**

## 解决方案

修改 `vendor/dongasai/dcat-admin2/src/Layout/Menu.php` 的 `isActive()` 方法，添加**前缀匹配**逻辑：

```php
$menuPath = trim($this->getPath($item['uri']), '/');

// 精确匹配或前缀匹配（支持 /create、/edit 等子路径）
return $menuPath == $path || str_starts_with($path, $menuPath . '/');
```

## 测试验证

### 单元测试

```
菜单 URI: user-agents

路径: admin/user-agents -> 激活: ✅ 是
路径: admin/user-agents/create -> 激活: ✅ 是
路径: admin/user-agents/1/edit -> 激活: ✅ 是
路径: admin/user-agents/1 -> 激活: ✅ 是
路径: admin/other -> 激活: ❌ 否
```

边界测试确保不会误激活其他菜单（如 `user-agents` 不会激活 `user` 或 `user-agent`）。

### 浏览器测试

访问 `http://192.168.4.107:32126/admin/user-agents/create`，左侧菜单 "User-Agent规则" 正确显示 `active` 类。

## 影响范围

此修改影响所有 dcat-admin 的 AdminController 子路径页面：
- `/create` - 创建页面
- `/{id}/edit` - 编辑页面
- `/{id}` - 详情页面

所有这些页面现在都能正确激活对应的菜单项。
