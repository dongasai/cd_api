# 修复 dcat-admin 菜单相关优化

## 问题一：子路径菜单无法激活

### 问题描述

访问 `admin/user-agents/create` 页面时，左侧菜单中的 `user_agents` 菜单项无法激活（不高亮）。

### 问题原因

dcat-admin 的 `Menu::isActive()` 方法使用**精确匹配**来判断菜单是否激活：

```php
// 原代码
return trim($this->getPath($item['uri']), '/') == $path;
```

这导致：
- 菜单 URI: `user-agents` → 解析为 `admin/user-agents`
- 当前路径: `admin/user-agents/create`
- 精确比较: `admin/user-agents` ≠ `admin/user-agents/create` → **菜单不激活**

### 解决方案

修改 `vendor/dongasai/dcat-admin2/src/Layout/Menu.php`：

1. **添加前缀匹配逻辑**（支持 `/create`、`/edit` 等子路径）

2. **新增外部设置机制**：

```php
use Dcat\Admin\Layout\Menu;

// 方式1: 通过菜单 ID 设置
Menu::setActiveId(76);

// 方式2: 通过菜单路径设置
Menu::setActivePath('user-agents');

// 重置（恢复默认行为）
Menu::resetActive();
```

## 问题二：手风琴菜单只能展开一个

### 问题描述

左侧菜单默认行为是手风琴模式：展开一个菜单会自动折叠其他菜单，无法同时展开多个。

### 解决方案

修改 `vendor/dongasai/dcat-admin2/resources/views/partials/sidebar.blade.php`：

```blade
{{-- 原代码 --}}
data-widget="treeview"

{{-- 修改后 --}}
data-widget="treeview" data-accordion="false"
```

添加 `data-accordion="false"` 属性即可禁用手风琴效果，允许多个菜单同时展开。

## 问题三：默认展开多个菜单

### 问题描述

希望页面加载时，某些菜单默认处于展开状态。

### 解决方案

修改 `vendor/dongasai/dcat-admin2/src/Layout/Menu.php`，新增 API：

```php
use Dcat\Admin\Layout\Menu;

// 设置默认展开的菜单 ID 列表
Menu::setDefaultExpandedIds([56, 57, 58]);

// 添加单个默认展开菜单
Menu::addDefaultExpandedId(56);

// 获取当前设置
Menu::getDefaultExpandedIds();

// 重置
Menu::resetDefaultExpanded();
```

### 使用场景

在中间件或控制器中设置：

```php
// 在 App\Http\Middleware 中
public function handle($request, Closure $next)
{
    // 设置默认展开的菜单
    \Dcat\Admin\Layout\Menu::setDefaultExpandedIds([56, 57]);

    return $next($request);
}
```

## 修改文件列表

1. `vendor/dongasai/dcat-admin2/src/Layout/Menu.php`
   - 添加前缀匹配逻辑
   - 添加 `setActiveId()` / `setActivePath()` 外部设置激活菜单
   - 添加 `setDefaultExpandedIds()` 默认展开菜单

2. `vendor/dongasai/dcat-admin2/resources/views/partials/menu.blade.php`
   - 使用 `isOpen()` 方法判断菜单展开状态

3. `vendor/dongasai/dcat-admin2/resources/views/partials/sidebar.blade.php`
   - 添加 `data-accordion="false"` 禁用手风琴效果

## 影响范围

此修改影响所有 dcat-admin 的 AdminController 子路径页面：
- `/create` - 创建页面
- `/{id}/edit` - 编辑页面
- `/{id}` - 详情页面

所有这些页面现在都能正确激活对应的菜单项。
