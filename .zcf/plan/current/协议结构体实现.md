# 协议结构体实现计划

> 创建时间: 2026-03-22
> 状态: 待批准

## 需求概述

实现 Protocol 层的协议结构体，提供：
- 强类型定义
- 严格验证（符合官方 API 规范）
- JSON 序列化
- 与 Shared\DTO 双向转换

## 架构设计

### 结构体定位与复用

**核心原则**：Protocol 层结构体描述的是 OpenAI/Anthropic API 格式，Provider 层直接复用这些结构体。

| 结构体 | 定义位置 | 使用方 |
|--------|----------|--------|
| `Protocol\OpenAI\ChatCompletionRequest` | Protocol 层 | Protocol Driver + OpenAI Provider |
| `Protocol\OpenAI\ChatCompletionResponse` | Protocol 层 | Protocol Driver + OpenAI Provider |
| `Protocol\Anthropic\MessagesRequest` | Protocol 层 | Protocol Driver + Anthropic Provider |
| `Protocol\Anthropic\MessagesResponse` | Protocol 层 | Protocol Driver + Anthropic Provider |

**命名空间在 Protocol 层的原因**：结构体本质是协议格式的类型定义，属于协议规范范畴。

### 非流式请求流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           客户端请求 (JSON)                                   │
│  POST /v1/chat/completions                                                   │
│  {"model": "gpt-4", "messages": [...], "temperature": 0.7}                  │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ HTTP Request
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Protocol Driver (入口)                                │
│  OpenAiChatCompletionsDriver::parseRequest()                                │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ ChatCompletionRequest::fromArray($data)
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Protocol\OpenAI\ChatCompletionRequest                     │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ 步骤 1: 类型转换                                                      │    │
│  │   - JSON 数组 → 强类型属性                                            │    │
│  │   - $data['model'] → $this->model (string)                           │    │
│  │   - $data['messages'] → Message[] 对象数组                            │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ 步骤 2: 验证                                                          │    │
│  │   - model: required|string                                           │    │
│  │   - messages: required|array|min:1                                   │    │
│  │   - temperature: nullable|numeric|between:0,2                        │    │
│  │   - 验证失败抛出 ValidationException                                  │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ $request->toSharedDTO()
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Shared\DTO\Request                                  │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ 协议无关的中间层，用于跨协议转换                                        │    │
│  │   - model: string                                                     │    │
│  │   - messages: Message[] (Shared\DTO\Message)                         │    │
│  │   - temperature: ?float                                               │    │
│  │   - maxTokens: ?int                                                   │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ Provider 处理
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Provider Layer                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ AbstractProvider::send()                                              │    │
│  │   1. ChatCompletionRequest::fromSharedDTO($request) ← 复用结构体      │    │
│  │   2. $apiRequest->toArray() → 上游请求格式                            │    │
│  │   3. HTTP POST → 上游 API                                             │    │
│  │   4. ChatCompletionResponse::fromArray($data) ← 复用结构体            │    │
│  │   5. $response->toSharedDTO() → 返回                                  │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ 返回 Shared\DTO\Response
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Shared\DTO\Response                                  │
│  - id: string                                                               │
│  - model: string                                                            │
│  - choices: Choice[]                                                        │
│  - usage: ?Usage                                                            │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ ChatCompletionResponse::fromSharedDTO($dto)
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                   Protocol\OpenAI\ChatCompletionResponse                     │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ 步骤 1: 类型转换                                                      │    │
│  │   - Shared\DTO\Response → 强类型属性                                  │    │
│  │   - $dto->choices → Choice[] 对象数组                                 │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ 步骤 2: 序列化                                                        │    │
│  │   - toArray() → OpenAI API 格式数组                                   │    │
│  │   - toJson() → JSON 字符串                                            │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ $response->toArray()
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                           客户端响应 (JSON)                                   │
│  {                                                                           │
│    "id": "chatcmpl-xxx",                                                     │

│    "model": "gpt-4",                                                         │
│    "choices": [...],                                                         │
```

#### 流式响应流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     上游 SSE 流 (Anthropic 格式示例)                          │
│  event: content_block_delta                                                 │
│  data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta",│
│         "text":"Hello"}}                                                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ Provider::parseStreamChunk()
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Shared\DTO\StreamChunk                                │
│  - contentDelta: "Hello"                                                    │
│  - type: StreamEventType::ContentDelta                                      │
│  - index: 0                                                                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ OpenAI 格式客户端需要转换
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                   Protocol\OpenAI\StreamChunkBuilder                         │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ fromSharedStreamChunk(StreamChunk)                                    │    │
│  │   - 转换事件类型: content_block_delta → chat.completion.chunk        │    │
│  │   - 构建 delta 结构: {"content": "Hello"}                            │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ toJson()                                                              │    │
│  │   → SSE 格式输出                                                      │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                     客户端 SSE 流 (OpenAI 格式)                               │
│  data: {"id":"chatcmpl-xxx","object":"chat.completion.chunk",               │
│         "choices":[{"delta":{"content":"Hello"},"index":0}]}                │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 阶段 2：实现 OpenAI 协议结构体

**目录**: `laravel/app/Services/Protocol/Driver/OpenAI/`

#### 2.1 ChatCompletionRequest.php

**职责**：OpenAI Chat Completions API 请求结构体

**主要属性**:
- model: string (必需)
- messages: Message[] (必需)
- temperature, top_p, max_tokens 等可选参数
- tools, tool_choice 工具调用参数
- response_format, seed 等高级参数

**验证规则** (符合 OpenAI API 规范):
- model: required|string
- messages: required|array|min:1
- messages.*.role: required|in:system,user,assistant,tool
- temperature: nullable|numeric|between:0,2
- top_p: nullable|numeric|between:0,1
- 等其他参数验证

**转换方法**:
- toSharedDTO(): 转换为协议无关的 Request DTO
- fromSharedDTO(): 从 Request DTO 构建

#### 2.2 Message.php

**职责**：OpenAI 消息结构体

**主要属性**:
- role: string (system|user|assistant|tool)
- content: string|array|null (支持多模态)
- tool_calls: ToolCall[]|null
- tool_call_id: string|null

#### 2.3 Tool.php

**职责**：OpenAI 工具定义结构体

**主要属性**:
- type: string (默认 'function')
- function: FunctionDef

#### 2.4 FunctionDef.php

**职责**：OpenAI 函数定义结构体

**主要属性**:
- name: string (必需, 最长64字符)
- description: string|null
- parameters: array|null (JSON Schema)
- strict: bool|null

#### 2.5 ChatCompletionResponse.php

**职责**：OpenAI Chat Completions API 响应结构体

**主要属性**:
- id: string
- object: string (默认 'chat.completion')
- created: int
- model: string
- choices: Choice[]
- usage: Usage|null
- system_fingerprint: string|null

#### 2.6 Choice.php

**职责**：OpenAI 响应选择结构体

**主要属性**:
- index: int
- message: Message|null (非流式)
- delta: Message|null (流式)
- finish_reason: string|null

#### 2.7 Usage.php

**职责**：OpenAI Token 使用量结构体

**主要属性**:
- prompt_tokens: int
- completion_tokens: int
- total_tokens: int
- prompt_tokens_details: array|null
- completion_tokens_details: array|null

### 阶段 3：实现 Anthropic 协议结构体

**目录**: `laravel/app/Services/Protocol/Driver/Anthropic/`

#### 3.1 MessagesRequest.php

**职责**：Anthropic Messages API 请求结构体

**主要属性**:
- model: string (必需)
- messages: Message[] (必需)
- max_tokens: int (必需, 范围 1-128000)
- system: string|array|null (独立字段)
- temperature, top_p, top_k 等可选参数
- tools, tool_choice 工具调用参数
- metadata 元数据

**验证规则** (符合 Anthropic API 规范):
- model: required|string
- messages: required|array|min:1
- max_tokens: required|integer|min:1|max:128000
- temperature: nullable|numeric|between:0,1
- 等其他参数验证

#### 3.2 Message.php

**职责**：Anthropic 消息结构体

**主要属性**:
- role: string (user|assistant)
- content: ContentBlock[]|string

#### 3.3 ContentBlock.php

**职责**：Anthropic 内容块结构体

**主要属性**:
- type: string (text|image|tool_use|tool_result|thinking)
- text: string|null
- source: array|null (图片源)
- id, name, input: 工具调用相关
- tool_use_id, content, is_error: 工具结果相关
- thinking, signature: 推理相关

#### 3.4 Tool.php

**职责**：Anthropic 工具定义结构体

**主要属性**:
- name: string
- description: string|null
- input_schema: array (JSON Schema)

#### 3.5 MessagesResponse.php

**职责**：Anthropic Messages API 响应结构体

**主要属性**:
- id: string
- type: string (默认 'message')
- role: string (默认 'assistant')
- content: ContentBlock[]
- model: string
- stop_reason: string|null
- stop_sequence: string|null
- usage: Usage

#### 3.6 Usage.php

**职责**：Anthropic Token 使用量结构体

**主要属性**:
- input_tokens: int
- output_tokens: int
- cache_creation_input_tokens: array|null
- cache_read_input_tokens: array|null
